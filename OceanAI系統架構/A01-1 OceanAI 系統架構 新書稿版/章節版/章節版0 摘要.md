# **Executive Summary｜執行摘要**

---

## **System Overview｜系統總覽**

**EN**  
OceanAI is a system-level intelligence architecture for spatial systems. It transforms fragmented devices into a coordinated operational system in which behavior emerges from coordination rather than isolated control.

**ZH-TW**  
OceanAI 是面向空間系統的系統級智慧架構。它將分散設備轉化為協同運作的整體，使系統行為源於協同，而非單點控制。

---

## **Core Architecture｜核心架構**

**EN**  
OceanAI adopts an **Islands + Swarm Runtime Architecture**: six coordinated layers—Cloud Intelligence, Edge Coordination, Hub Local Runtime, Gateway Access, Synchronization, and Device Execution—form a distributed runtime system. **OceanAI Runtime (ORT)** coordinates continuous operation across layers; **ArtiOS Runtime (ART)** implements ORT locally inside Artibird environments (Chapter 8 — ArtiOS Runtime Layer｜第八章｜ArtiOS Runtime 層). Data flows as state and events; control follows Sense → Interpret → Coordinate → Act → Learn.

```text
OceanAI       = system-level intelligence architecture
ORT           = OceanAI Runtime / system-level runtime architecture
ART           = ArtiOS Runtime / local runtime implementation
Artibird      = physical device ecosystem
```

**ZH-TW**  
OceanAI 採用 **群島式群集 Runtime 架構（Islands + Swarm Runtime Architecture）**：六層協同——雲端智慧、邊緣協同、中樞本地 Runtime、網關接入、同步接入與設備執行——構成分布式 Runtime 系統。**OceanAI Runtime（ORT）** 協調跨層持續運行；**ArtiOS Runtime（ART）** 在 Artibird 環境中本地實現 ORT（Chapter 8 — ArtiOS Runtime Layer｜第八章｜ArtiOS Runtime 層）。資料以狀態與事件流動；控制遵循 Sense → Interpret → Coordinate → Act → Learn。

```text
OceanAI       = 系統級智慧架構
ORT           = OceanAI Runtime／系統級 Runtime 架構
ART           = ArtiOS Runtime／本地 Runtime 實現
Artibird      = 實體設備生態
```

---

## **Intelligence, Control & Operations｜智慧、控制與運作**

**EN**  
The Intelligence Layer connects rule, context, and swarm intelligence with AI-assisted learning, runtime feedback, and governance (Chapter 7 — Intelligence Layer｜第七章｜智慧層). Control abstracts user input into intent (Chapter 9 — Control Architecture｜第九章｜控制架構). Artibird products carry runtime through Hub, Gate, Sync, and edge nodes (Chapter 10 — Artibird Product Ecosystem｜第十章｜Artibird 產品生態). Deployments are built from coordinated runtime islands (Chapter 11 — Deployment Scenarios｜第十一章｜部署場景). Security and governance keep islands locally controllable and cross-island behavior auditable (Chapter 12 — Security & Governance｜第十二章｜安全與治理).

**ZH-TW**  
智慧層連接規則、情境與群集智慧，以及 AI 輔助學習、Runtime 回饋與治理（Chapter 7 — Intelligence Layer｜第七章｜智慧層）。控制架構將使用者輸入抽象為意圖（Chapter 9 — Control Architecture｜第九章｜控制架構）。Artibird 產品透過 Hub、Gate、Sync 與邊緣節點承載 Runtime（Chapter 10 — Artibird Product Ecosystem｜第十章｜Artibird 產品生態）。部署以協同 Runtime 島嶼為單元（Chapter 11 — Deployment Scenarios｜第十一章｜部署場景）。安全與治理確保島嶼本地可控、跨島行為可審計（Chapter 12 — Security & Governance｜第十二章｜安全與治理）。

## **System Value｜系統價值**

**EN**  
OceanAI shifts the paradigm from device connectivity to system coordination—from direct control to coordinated behavior, and from automation to intelligence as a property of the system.

**ZH-TW**  
OceanAI 將範式從設備連接轉向系統協同，從直接控制轉向協同行為，從自動化轉向以系統為載體的智慧。
