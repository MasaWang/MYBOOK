## **A.1 Core System Terms｜核心系統術語**

---

### **System｜系統**

**EN**  
A structured set of components, interactions, and rules that operate as a continuous entity to produce coordinated behavior.

**ZH-TW**  
由組件、交互與規則構成的結構體，作為連續整體運作以產生協同行為。

---

### **Architecture｜架構**

**EN**  
The structural design that defines how system components are organized, interact, and operate.

**ZH-TW**  
定義系統組件如何組織、交互與運作的結構設計。

---

### **Layer｜層**

**EN**  
A logical grouping of system functions with defined responsibilities and interaction boundaries.

**ZH-TW**  
具備明確責任與交互邊界的系統功能分層。

---

### **Domain｜域**

**EN**  
A functional partition of the system defined by its operational role, such as Cloud, Edge, Device, or the ArtiOS Runtime local runtime boundary between Edge coordination and physical execution.

**ZH-TW**  
依運作角色劃分的系統區域，如雲端、邊緣、設備，或 Edge 協同與物理執行之間的 ArtiOS Runtime 本地 Runtime 邊界。

---

### **Node｜節點**

**EN**  
A system component that performs a defined role within sensing, coordination, control, or execution.

**ZH-TW**  
在感知、協同、控制或執行中承擔特定角色的系統組件。

---

### **Islands + Swarm Runtime Architecture｜群島式群集 Runtime 架構**

**EN**  
The core architectural model of OceanAI in which each local environment operates as an autonomous runtime island, while islands and their internal devices cooperate as distributed participants in a larger runtime field.

**ZH-TW**  
OceanAI 的核心架構模型：每個本地環境作為自治 Runtime 島嶼運行，島嶼與其內部設備在更大的 Runtime 場中作為分布式參與者協同。

---

### **Runtime Island｜Runtime 島嶼**

**EN**  
A local environment—such as a room, home, classroom, floor, or campus zone—that maintains local sensing, control, rules, scenes, state, and response capability as an autonomous runtime unit.

**ZH-TW**  
具備本地感知、控制、規則、場景、狀態與回應能力的本地環境（如房間、家庭、教室、樓層或校園區域），作為自治 Runtime 單元運行。

---

### **Swarm Runtime Field｜群集 Runtime 場**

**EN**  
The distributed operational field formed by runtime islands and their participating devices, kept coordinated through gateways, hubs, cloud services, and coordination protocols.

**ZH-TW**  
由 Runtime 島嶼與其參與設備構成的分布式運作場，透過網關、中樞、雲端服務與協同協議保持協同。

---

### **Swarm Intelligence｜群集智能**

**EN**  
In the OceanAI architecture, the cooperative behavior model by which runtime islands and their internal devices interact as distributed participants. This term refers to system-level coordination within the Islands + Swarm Runtime Architecture, not to standalone multi-agent AI systems.

**ZH-TW**  
在 OceanAI 架構中，Runtime 島嶼與其內部設備作為分布式參與者共同運作的協同行為模型。此術語指群島式群集 Runtime 架構內的系統級協同，而非獨立的多 Agent AI 系統。

## **A.4 Runtime Terms｜ORT / ART Runtime 術語**

---

### **OceanAI Runtime (ORT)｜OceanAI Runtime（ORT）**

**EN**  
OceanAI Runtime (ORT) is the system-level runtime architecture of OceanAI. It coordinates continuous operation across cloud intelligence, edge coordination, local runtime implementation, and device execution. ORT is not cloud-only—it governs runtime flow, system state, learning, and governance across Cloud, Edge, local runtime, and Device layers.

**ZH-TW**  
OceanAI Runtime（ORT）是 OceanAI 的系統級 Runtime 架構。它協調雲端智慧、邊緣協同、本地 Runtime 實現與設備執行之間的持續運行。ORT 不是僅限雲端的 Runtime 核心——它橫跨 Cloud、Edge、本地 Runtime 與 Device 各層，治理 Runtime 流程、系統狀態、學習與治理。

---

### **ArtiOS Runtime (ART)｜ArtiOS Runtime（ART）**

**EN**  
ArtiOS Runtime (ART) is the local runtime implementation of OceanAI Runtime inside Artibird environments. It makes ORT locally executable through local rules, scenes, device states, events, protocol abstraction, offline fallback, and runtime logs. ART runs primarily through Artibird Hub and may extend selected capabilities through Gate, Sync, and other local runtime devices.

**ZH-TW**  
ArtiOS Runtime（ART）是 OceanAI Runtime 在 Artibird 環境中的本地 Runtime 實現。它透過本地規則、場景、設備狀態、事件、協議抽象、離線備援與 Runtime 日誌，使 ORT 在本地可執行。ART 主要透過 Artibird Hub 運行，部分能力可延伸至 Gate、Sync 與其他本地 Runtime 設備。

---

### **Runtime Pair｜Runtime 雙層結構**

**EN**  
ORT and ART form a system-to-local runtime pair. ORT defines the system-level runtime architecture, while ART implements that runtime locally inside Artibird environments.

**ZH-TW**  
ORT 與 ART 構成系統到本地的 Runtime 雙層結構。ORT 定義系統級 Runtime 架構，ART 則在 Artibird 環境中本地實現這一 Runtime。

---

### **Runtime｜Runtime（架構方法）**

**EN**  
The architecture of continuous operation in OceanAI, formally expressed as OceanAI Runtime (ORT). In general usage, *runtime* describes how intelligent spaces remain coordinated, responsive, and continuously operable—not a product, operating system, or hardware node.

**ZH-TW**  
OceanAI 中持續運行的架構方法，正式表述為 OceanAI Runtime（ORT）。一般用法中，*Runtime* 描述智慧空間如何保持協同、回應與連續運行——不是產品、作業系統或硬體節點。

### **ArtiOS Runtime Layer｜ArtiOS Runtime 層**

**EN**  
The chapter defined in Chapter 8 — ArtiOS Runtime Layer｜第八章｜ArtiOS Runtime 層. ART is the local implementation carried within this layer.

**ZH-TW**  
Chapter 8 — ArtiOS Runtime Layer｜第八章｜ArtiOS Runtime 層 所定義的章節主題。ART 是在此層中承載的本地實現。

---

### **ArtiOS Runtime Boundary｜ArtiOS Runtime 邊界**

**EN**  
The system-model boundary between Edge coordination and physical device execution. ArtiOS Runtime defines where local runtime execution, local state, local rules, and offline continuity live inside an Artibird environment.

**ZH-TW**  
Edge 協同與物理設備執行之間的系統模型邊界。ArtiOS Runtime 定義本地 Runtime 執行、本地狀態、本地規則與離線連續性在 Artibird 環境中的位置。

**ZH-TW**

---

## **B.1 Principle of Local-First Operation｜本地優先原則**

**EN**  
Core system functions are executed locally to ensure deterministic behavior, low latency, and operational continuity independent of external connectivity.

Cloud services extend system capabilities but are not required for fundamental operation.

**ZH-TW**  
核心系統功能在本地執行，以確保確定性行為、低延遲與在無外部連接情況下的持續運作能力。

雲端用於擴展能力，而非維持基本運作。

---

## **B.2 Principle of State-Centric Design｜狀態中心原則**

**EN**  
System behavior is derived from state rather than direct command relationships. All decisions and actions are evaluated based on a continuously updated system state.

State serves as the primary reference for coordination.

**ZH-TW**  
系統行為以狀態為核心，而非直接指令關係。所有決策與行為均基於持續更新的系統狀態進行判斷。

狀態是協同運作的核心依據。

---

## **B.3 Principle of Event-Driven Execution｜事件驅動原則**

**EN**  
System operation is structured around events triggered by user interaction, environmental changes, or internal conditions.

Events initiate processing, coordination, and execution across the system.

**ZH-TW**  
系統運作以事件為核心結構，事件由使用者操作、環境變化或內部條件觸發。

事件驅動系統的處理、協同與執行。

## **B.4 Principle of Separation of Concerns｜關注分離原則**

**EN**  
System functions are divided across layers and domains with clearly defined responsibilities.

Data processing, decision-making, and execution are separated to ensure modularity and scalability.

**ZH-TW**  
系統功能依層與域進行劃分，並具備明確責任邊界。

資料處理、決策與執行相互分離，以確保模組化與可擴展性。

---

## **B.5 Principle of Distributed Intelligence｜分布式智慧原則**

**EN**  
Intelligence is distributed across system layers rather than centralized.

Rule-based logic, coordination mechanisms, and AI inference operate together to produce system behavior.

**ZH-TW**  
智慧分布於系統各層，而非集中於單一位置。

規則邏輯、協同機制與 AI 推理共同形成系統行為。

---

## **B.6 Principle of Deterministic Execution｜確定性執行原則**

**EN**  
Execution at the Device Execution Layer is deterministic and predictable.

**ZH-TW**  
設備執行層的執行保持確定性與可預測性。

系統的靈活性來自協同與邏輯，而非執行的不確定性。

---

System interaction is independent of underlying communication protocols.

**ZH-TW**  
系統交互不依賴底層通訊協議。

不同設備透過統一交互模型整合於系統中。

## **B.8 Principle of Continuous Operation｜連續運作原則**

**EN**  
The system operates as a continuous process rather than discrete interactions.

State, events, and decisions flow without interruption across system layers.

**ZH-TW**  
系統以連續過程運作，而非離散交互。

狀態、事件與決策在各層之間持續流動。

## **B.9 Principle of System-Oriented Design｜系統導向設計原則**

**EN**  
Design focuses on system behavior rather than individual device functionality.

**ZH-TW**  
設計以系統行為為核心，而非單一設備功能。

設備作為協同系統中的節點存在。

System behavior must remain observable, controllable, and auditable.

Security, access control, and policy enforcement are integrated into system design.

**ZH-TW**  
系統行為必須具備可觀測、可控制與可審計能力。

安全、權限控制與策略機制為設計的一部分。
