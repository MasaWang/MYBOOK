# 0.2 Preface

# 0.2 前言

## EN

The OceanAI System Architecture Specification v1.1 defines the architecture-layer foundation of OceanAI as a system-level intelligence architecture for coordinated environments.

This document is written after the completion of the OceanAI / Artibird System Specification v1.0. The v1.0 document defines how OceanAI and Artibird operate together as a coordinated intelligent environment system. This v1.1 document moves one level higher. It defines OceanAI as an architecture: how it is organized, how it operates, how it maintains runtime continuity, how it coordinates local, cloud, and edge intelligence, how it governs data and AI-assisted behavior, and how its system claims must remain within validated boundaries.

OceanAI is not defined here as a single product, device, hub, gateway, application, or cloud service. It is defined as a system architecture that organizes physical devices, runtime services, data models, local intelligence, cloud coordination, edge interpretation, security, privacy, validation, and governance into one coherent structure.

The purpose of this document is to establish the design logic behind OceanAI. It explains why the architecture is local-first, why cloud services enhance rather than replace local runtime behavior, why edge intelligence is bounded by validation and governance, and why coordinated intelligence must be understood as a runtime condition rather than a collection of isolated device functions.

This specification is intended to guide future architecture work, product alignment, software planning, edge and cloud development, data model design, validation strategy, deployment planning, and commercial claim governance.

The document should be read as an architecture specification, not as a product specification. Individual products may participate in the OceanAI architecture, but this document does not define the detailed hardware, firmware, commercial positioning, or SKU structure of those products. Those details remain in product engineering specifications and commercial specifications.

## ZH-TW

《OceanAI 系統架構規格書 v1.1》定義 OceanAI 作為面向協同空間的系統級智慧架構之架構層基礎。

本文件在《OceanAI / Artibird 系統規格書 v1.0》完成後撰寫。v1.0 文件定義 OceanAI 與 Artibird 如何作為一個協同智慧空間系統共同運作。v1.1 文件則上升一個層級，定義 OceanAI 作為一套架構：它如何被組織、如何運作、如何維持 Runtime 延續性、如何協同本地、雲端與邊緣智慧、如何治理數據與 AI 輔助行為，以及其系統宣稱如何保持在已驗證邊界內。

在本文件中，OceanAI 不被定義為單一產品、設備、中樞、網關、應用程式或雲端服務。它被定義為一套系統架構，用於將實體設備、Runtime 服務、數據模型、本地智慧、雲端協同、邊緣解讀、安全、隱私、驗證與治理組織為一個連貫結構。

本文件的目的，是建立 OceanAI 背後的設計邏輯。它說明為什麼該架構以本地優先作為預設原則，為什麼雲端服務是增強而非取代本地 Runtime 行為，為什麼邊緣智慧必須受到驗證與治理約束，以及為什麼協同智慧應被理解為一種 Runtime 條件，而不是一組孤立設備功能的集合。

本規格書用於指導未來架構工作、產品對齊、軟體規劃、邊緣與雲端開發、數據模型設計、驗證策略、部署規劃與商業宣稱治理。

本文件應被視為架構規格書，而不是產品規格書。個別產品可以參與 OceanAI 架構，但本文件不定義這些產品的詳細硬體、韌體、商業定位或 SKU 結構。這些內容應保留在產品工程規格書與商業規格書中。
