# 8. Event, Command, and State Model

# 8. 事件、指令與狀態模型

---

## 8.1 Purpose

## 8.1 目的

## EN

This chapter defines the Event, Command, and State Model of OceanAI. It describes how signals, instructions, and system states are represented, transmitted, and processed within the runtime architecture.

The model ensures that events from devices, user inputs, cloud commands, Edge interpretations, and automation rules are handled consistently. It defines the priority, validation, synchronization, and failure behavior of all events and commands in the coordinated intelligent environment.

## ZH-TW

本章定義 OceanAI 的事件、指令與狀態模型。它描述信號、指令與系統狀態在 Runtime 架構中如何表示、傳輸與處理。

此模型確保來自設備、用戶輸入、雲端指令、Edge 解讀與自動化規則的事件被一致處理。它定義協同智慧環境中所有事件與指令的優先級、驗證、同步與故障行為。

## 8.2 Event Model

## 8.2 事件模型

## EN

Events represent changes in the system or environment that may trigger actions. Events may originate from:

- **Device events:** Sensor updates, relay state changes, energy measurements, appliance feedback.
    
- **User events:** Switch touches, Panel interactions, App commands.
    
- **Cloud events:** Remote commands, account updates, lifecycle notifications.
    
- **Edge events:** AI-assisted interpretations, recommendations, detection signals.
    
- **System events:** Runtime heartbeat, error reports, validation alerts.
    

Each event includes an **event type, source, timestamp, payload, priority, security metadata, and validation status**.

## ZH-TW

事件表示系統或環境中的變化，可能觸發動作。事件可能來源於：

- **設備事件：** 感測器更新、繼電器狀態變化、能源測量、家電反饋。
    
- **用戶事件：** Switch 觸控、Panel 操作、App 指令。
    
- **雲端事件：** 遠端指令、帳號更新、生命週期通知。
    
- **Edge 事件：** AI 輔助解讀、建議、偵測信號。
    
- **系統事件：** Runtime 心跳、錯誤報告、驗證警報。
    

每個事件包含 **事件類型、來源、時間戳、內容載荷、優先級、安全元數據與驗證狀態**。

## 8.3 Command Model

## 8.3 指令模型

## EN

Commands represent actionable instructions delivered to devices or runtime services. Commands may originate from:

- **User commands:** Local interactions through Switch, Panel, or App.
    
- **Automation commands:** Triggered by runtime rules, sensor thresholds, or schedules.
    
- **Cloud commands:** Remote instructions issued from cloud services.
    
- **Edge-assisted commands:** Derived from Edge AI recommendations or interpretations.
    

Each command includes **command type, target, parameters, priority, timestamp, and validation metadata**. Commands must respect runtime priority rules, access permissions, safety constraints, and execution boundaries.

## ZH-TW

指令表示傳送給設備或 Runtime 服務的可執行指示。指令可能來源於：

- **用戶指令：** 透過 Switch、Panel 或 App 的本地互動。
    
- **自動化指令：** 由 Runtime 規則、感測器閾值或排程觸發。
    
- **雲端指令：** 由雲端服務發出的遠端操作指令。
    
- **Edge 輔助指令：** 由 Edge AI 推薦或解讀產生。
    

每個指令包含 **指令類型、目標、參數、優先級、時間戳與驗證元數據**。指令必須遵循 Runtime 優先級規則、存取權限、安全約束與執行邊界。

|Command Type|Description|
|---|---|
|User|Generated by direct user interaction|
|Automation|Generated by runtime rule engine or automation scheduler|
|Cloud|Generated remotely by cloud services|
|Edge-assisted|Generated by AI-assisted interpretation or recommendation|

## 中文表

|指令類型|說明|
|---|---|

|自動化指令|由 Runtime 規則引擎或自動化排程生成|
|雲端指令|由雲端服務遠端生成|
|Edge 輔助指令|由 AI 輔助解讀或推薦生成|

## 8.4 State Model

## 8.4 狀態模型

## EN

The State Model defines the current condition of devices, services, spaces, and the system itself. States may include:

- **Device state:** On/off, brightness, mode, energy level, error status.
    
- **Scene state:** Active, inactive, paused, triggered.
    
- **Automation state:** Enabled, disabled, executing, queued.
    
- **System state:** Online, offline, degraded, error, updating.
    
- **User context state:** Permissions, role, presence, activity.
    

State changes are triggered by events or commands, evaluated against automation rules, and may propagate across runtime, cloud, and Edge layers.

## ZH-TW

狀態模型定義設備、服務、空間與系統本身的當前條件。狀態可能包括：

- **設備狀態：** 開/關、亮度、模式、能量等級、錯誤狀態。
    
- **場景狀態：** 啟用、停用、暫停、已觸發。
    
- **自動化狀態：** 啟用、停用、執行中、排隊中。
    
- **系統狀態：** 在線、離線、降級、錯誤、更新中。
    
- **用戶上下文狀態：** 權限、角色、在場狀態、活動狀態。
    

狀態變化由事件或指令觸發，經過自動化規則判斷，可能跨 Runtime、雲端與 Edge 層級傳播。

## 8.5 Event-Command-State Flow

## 8.5 事件—指令—狀態流

## EN

The Event-Command-State Flow defines how events trigger commands, how commands affect states, and how states propagate within the runtime, cloud, and Edge layers.

- **Event → Command:** Events from devices, users, or Edge may generate commands.
    
- **Command → State:** Commands cause state changes in devices, scenes, automation, or system context.
    
- **State → Event:** State changes may generate new events, trigger automations, or notify cloud/Edge systems.
    

The flow ensures coordinated operation, enforces priority rules, and maintains consistency across all layers.

## ZH-TW

事件—指令—狀態流定義事件如何觸發指令，指令如何影響狀態，以及狀態如何在 Runtime、雲端與 Edge 層級中傳播。

- **事件 → 指令：** 來自設備、用戶或 Edge 的事件可能產生指令。
    
- **指令 → 狀態：** 指令導致設備、場景、自動化或系統上下文的狀態變化。
    
- **狀態 → 事件：** 狀態變化可能產生新事件、觸發自動化或通知雲端/Edge 系統。
    

此流程確保協同運作，執行優先級規則，並維持跨所有層級的一致性。
