## EN

This **OceanAI System Architecture Specification v1.1** is now completed as an architecture-level reference document for the OceanAI / Artibird coordinated intelligent environment system.

It defines OceanAI as a system-level intelligence architecture and clarifies how Local Runtime, Cloud services, Edge intelligence, device access, interaction, sensing, execution, data models, governance, validation, deployment, reliability, versioning, and claim boundaries operate as one coherent system.

This document is intended to serve as the architecture foundation above product specifications, commercial specifications, deployment references, validation records, and future system evolution documents.

It does not replace individual Artibird product engineering specifications. Instead, it defines how those products participate within the larger OceanAI architecture.

All implementation, capacity, reliability, certification, security, privacy, AI, energy, and commercial claims must remain tied to validated evidence, supported versions, deployment conditions, and approved boundary language.

OceanAI is therefore documented not as a collection of isolated products, but as a coordinated system architecture for spatial intelligence.
