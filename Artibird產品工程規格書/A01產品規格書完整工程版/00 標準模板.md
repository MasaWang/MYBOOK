# 4. Core Design Principles

# 4. 核心設計原則

## 4.1 Local-First Runtime Architecture

## 4.1 本地優先 Runtime 架構

**EN**  
The product should prioritize local execution, local responsiveness, and local service continuity wherever technically feasible. Core functions should remain available within the local environment when external cloud connectivity is unavailable, provided the deployment architecture supports local runtime operation.

**ZH-TW**  
產品應在技術可行範圍內優先保障本地執行、本地回應與本地服務持續性。當外部雲端連接不可用時，在部署架構支援本地 Runtime 運作的條件下，核心功能應盡可能維持於本地環境中可用。

## 4.2 Coordinated System Participation

## 4.2 協同式系統參與

**EN**  
The product should not be treated as an isolated endpoint. It should participate as a coordinated runtime entity within the Artibird ecosystem and the OceanAI system architecture.

**ZH-TW**  
產品不應被視為孤立終端，而應作為 Artibird 生態與 OceanAI 系統架構中的協同 Runtime 實體參與運作。

## 4.3 Cloud-Enhanced Intelligence

## 4.3 雲端增強智慧

**EN**  
Cloud services may provide remote control, synchronization, lifecycle management, analytics, multi-site coordination, and intelligent enhancement. Cloud functions should extend local operation rather than replace local runtime availability.

**ZH-TW**  
雲端服務可提供遠端控制、同步、生命週期管理、分析、多站點協同與智慧增強。雲端功能應擴展本地運作能力，而不是取代本地 Runtime 可用性。

## 4.4 Distributed Operational Resilience

## 4.4 分布式運作韌性

**EN**  
The product should support predictable behavior under unstable network conditions, device disconnection, power interruption, firmware recovery, and runtime load variation.

**ZH-TW**  
產品應在網路不穩定、設備斷線、電源中斷、韌體恢復與 Runtime 負載變化時，保持可預期的運作行為。

## 4.5 Security by Design

## 4.5 安全預設設計

**EN**  
Security should be embedded into the product architecture from the beginning. Authentication, authorization, encrypted communication, secure update, credential protection, and debug interface control should be defined as engineering requirements.

**ZH-TW**  
安全性應從產品架構初期即嵌入設計中。身份認證、授權、加密通訊、安全更新、憑證保護與 Debug 介面控制，皆應被定義為工程要求。

# 15. Engineering Claim Policy

## Mandatory Statement

## 強制聲明

**EN**  
All performance claims, especially concurrent connections, wireless range, processing capacity, automation capacity, battery life, reliability, and compatibility, must be based on validated test reports under defined conditions. Theoretical chip specifications shall not be used as product claims.

**ZH-TW**  
所有性能宣稱，尤其是併發連接數、無線距離、處理能力、自動化容量、電池壽命、可靠性與相容性，必須基於明確條件下的已驗證測試報告。晶片理論規格不得作為產品宣稱。
