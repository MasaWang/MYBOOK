# 3. Product Commercial Overview

# 3. 產品商業總覽

## EN

Artibird Hub is the local system hub and runtime center for the Artibird product ecosystem. Built on a Raspberry Pi 5-based local hub architecture, it provides the local service foundation for coordinated intelligent environments, including local messaging, device state management, automation logic, local web services, data synchronization paths, and ArtiOS-related runtime modules.

In homes, classrooms, offices, campuses, and commercial buildings, Artibird Hub allows supported Artibird devices to operate as part of one coordinated local system. Devices such as Artibird Gate, Switch, Panel, Act, Sense, Sync, and Vision can exchange events, synchronize state, trigger scenes, execute automations, and participate in shared runtime behavior through Hub-supported services.

Artibird Hub follows a local-first, cloud-enhanced model. Supported local functions are designed to remain available within the local environment when the required devices, network paths, and runtime services are present. When connected to OceanAI Cloud, the system may gain remote access, lifecycle management, account-level synchronization, multi-site visibility, analytics, software updates, and remote maintenance capabilities.

When paired with OceanAI Edge, Artibird Hub can also provide local system context for AI-assisted analysis, visual understanding, environmental optimization, and intelligent response. In this role, Hub becomes the practical operating center that allows Artibird devices to move from isolated control into coordinated system behavior.

Artibird Hub should not be presented as a generic smart home accessory. It is the local runtime foundation for building structured, expandable, and manageable intelligent environments.

## ZH-TW

Artibird Hub 是 Artibird 產品生態的本地系統中樞與 Runtime 中心。它基於 Raspberry Pi 5 本地中樞架構，為協同智慧空間提供本地服務基礎，包括本地消息通訊、設備狀態管理、自動化邏輯、本地 Web 服務、數據同步路徑與 ArtiOS 相關 Runtime 模組。

在住宅、教室、辦公室、校園與商業建築中，Artibird Hub 使受支援 Artibird 設備能夠作為同一個協同本地系統的一部分運作。Artibird Gate、Switch、Panel、Act、Sense、Sync、Vision 等設備可透過 Hub 支援的服務交換事件、同步狀態、觸發場景、執行自動化，並參與共享 Runtime 行為。

Artibird Hub 採用本地優先、雲端增強模式。在所需設備、網路路徑與 Runtime 服務具備的條件下，受支援本地功能設計為可在本地環境中保持可用。當接入 OceanAI Cloud 後，系統可進一步獲得遠端存取、生命週期管理、帳號級同步、多站點可視化、分析、軟體更新與遠端維護能力。

當與 OceanAI Edge 配合時，Artibird Hub 也可為 AI 輔助分析、視覺理解、環境優化與智慧回應提供本地系統上下文。在這一角色中，Hub 成為使 Artibird 設備從孤立控制走向協同系統行為的實際運作中心。

Artibird Hub 不應被表述為普通智慧家居配件。它是構建結構化、可擴展、可管理智慧環境的本地 Runtime 基礎。

## 4.3 Long Description

## 4.3 長描述

**EN**  
Artibird Hub is designed for users, installers, and organizations that need a reliable local runtime center rather than device-by-device control. It hosts or supports the local services that allow supported Artibird devices to communicate, synchronize, respond, and participate in coordinated operation.

In a complete Artibird deployment, Hub works with Artibird Gate, ArtiOS, OceanAI Edge, OceanAI Cloud, and the Artibird App to support local automation, system state management, device onboarding, scene execution, data flow, monitoring, and remote management where enabled. It is especially suitable for homes, classrooms, offices, campuses, and buildings where multiple devices need to operate as a coordinated environment.

Artibird Hub does not replace every device function, nor does it make unlimited system-capacity claims. Its commercial value depends on validated system scale, supported device categories, tested runtime services, configured network conditions, and defined deployment boundaries. This makes Hub suitable for commercial and project deployments that require clear planning, reliable local coordination, and long-term system expansion.

# 12. Commercial Claim Policy

# 12. 商業宣稱政策

## EN

All Artibird Hub commercial claims must remain consistent with the related engineering specification and validation results. Commercial materials must not use absolute claims such as “unlimited devices,” “fully autonomous,” “guaranteed uptime,” “works with all products,” or “complete offline operation” unless these claims are supported by formal validation evidence and clearly defined operating conditions.

Capacity-related and performance-related claims must reference validated test conditions and official validation reports. Terms such as “local runtime center,” “system coordination,” “stable local operation,” “multi-device support,” or “multi-site management” should only be used with clear boundaries, such as supported device categories, tested software versions, local network conditions, service configuration, storage limits, and validated system scale.
