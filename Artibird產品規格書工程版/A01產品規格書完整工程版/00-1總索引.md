# 3. 產品家族架構

## EN

The Artibird product family is organized as a distributed physical layer for the OceanAI coordinated intelligence architecture. Each product category provides a defined role in the runtime system, from local coordination and network access to sensing, interaction, execution, synchronization, and visual perception.

Artibird products should not be treated as isolated devices. They are coordinated runtime participants that allow OceanAI to operate across physical spaces.

# 6. Local-First and Cloud-Enhanced Policy

# 6. 本地優先與雲端增強政策

## EN

All Artibird product specifications must follow the same operating principle: core local behavior should remain available within the local environment whenever technically feasible, while cloud services should extend the system through remote access, synchronization, analytics, lifecycle management, model enhancement, and multi-site coordination.

Cloud functionality must be described as an enhancement layer, not as the sole operational foundation unless the product function explicitly requires cloud services.

## ZH-TW

所有 Artibird 產品規格書都必須遵循相同的運作原則：在技術可行條件下，核心本地行為應保持於本地環境中可用；雲端服務則透過遠端存取、同步、分析、生命週期管理、模型增強與多站點協同擴展系統能力。

除非產品功能明確需要雲端服務，否則雲端功能應被描述為增強層，而不是唯一運作基礎。

## English Table

|---|---|---|

|Vision|Local visual events, local metadata, Edge-assisted inference|Remote viewing, event history, backup|
|Synchronization|Local state, Hub coordination, device events|Cross-site sync, account sync, cloud analytics|
|Security|Local authentication, local permissions, secure pairing|Account management, remote audit, lifecycle policy|
|Maintenance|Local diagnostics, OTA where available|Remote update policy, fleet management|

# 9. Engineering Claim Control

# 9. 工程宣稱控制

## EN

All Artibird product specifications must apply the Engineering Claim Policy. Public claims must not be based solely on chip specifications, module datasheets, relay ratings, sensor datasheets, AI benchmark numbers, or theoretical communication limits.

Every claim involving range, capacity, latency, battery life, load rating, detection accuracy, compatibility, privacy behavior, security, or reliability must be supported by validation evidence under defined test conditions.

## ZH-TW

所有 Artibird 產品規格書都必須套用工程宣稱政策。公開宣稱不得僅基於晶片規格、模組資料手冊、繼電器額定值、感測器資料手冊、AI 基準分數或理論通訊上限。

凡涉及距離、容量、延遲、電池壽命、負載額定值、檢測精度、相容性、隱私行為、安全性或可靠性的宣稱，都必須在明確測試條件下取得驗證證據支持。

|Connection Capacity|System-level capacity validation|
