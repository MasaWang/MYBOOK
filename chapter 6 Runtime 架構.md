## 6.2 Runtime Definition

## 6.2 Runtime 定義

**EN**  
Runtime refers to the active operating state of the OceanAI / Artibird system, where devices, user interfaces, local services, cloud services, edge intelligence, events, states, commands, scenes, automations, and execution actions are processed as part of one coordinated system.

In this specification, runtime behavior is primarily maintained by Artibird Hub and ArtiOS. Artibird Gate provides device access. Artibird devices provide interaction, sensing, execution, synchronization, and visual context. OceanAI Cloud and OceanAI Edge participate where supported by deployment configuration.

**ZH-TW**  
Runtime 指 OceanAI / Artibird 系統的實際運行狀態。在此狀態中，設備、用戶介面、本地服務、雲端服務、邊緣智慧、事件、狀態、指令、場景、自動化與執行動作，作為一個協同系統的一部分被處理。

在本規格書中，Runtime 行為主要由 Artibird Hub 與 ArtiOS 維持。Artibird Gate 提供設備接入。Artibird 設備提供互動、感知、執行、同步與視覺上下文。OceanAI Cloud 與 OceanAI Edge 在部署配置支援時參與系統運作。

## 6.4 Local Service Hosting

## 6.4 本地服務承載

**EN**  
Local service hosting is primarily provided by Artibird Hub. Hub may host local runtime services, MQTT broker, state management services, automation modules, web services, device coordination services, and ArtiOS-related modules where supported.

Local service hosting allows supported system behavior to continue within the local environment when required local components are available.

**ZH-TW**  
本地服務承載主要由 Artibird Hub 提供。Hub 可在支援條件下承載本地 Runtime 服務、MQTT Broker、狀態管理服務、自動化模組、Web 服務、設備協同服務與 ArtiOS 相關模組。

本地服務承載使受支援系統行為在所需本地元件具備時，能夠在本地環境中繼續運作。

Events may include Switch touch events, Panel actions, App commands, Sense readings, Vision alerts, Act execution responses, Sync command results, device status changes, cloud commands, and edge-assisted interpretations.

事件處理定義系統如何接收與處理由受支援設備、用戶介面、雲端服務、邊緣服務與本地 Runtime 服務產生的事件。

事件可包括 Switch 觸控事件、Panel 操作、App 指令、Sense 讀數、Vision 警報、Act 執行回應、Sync 指令結果、設備狀態變化、雲端指令與 Edge 輔助解讀。

## 6.6 State Management

## 6.6 狀態管理

**EN**  
State management defines how the system maintains the current status of devices, rooms, scenes, automations, connections, runtime services, and system health.

Device states should be maintained locally where required components are present. Selected states may synchronize with OceanAI Cloud where supported. State accuracy depends on device connectivity, reporting behavior, runtime processing, and synchronization status.

**ZH-TW**  
狀態管理定義系統如何維護設備、房間、場景、自動化、連接、Runtime 服務與系統健康的當前狀態。

在所需元件具備時，設備狀態應在本地維護。特定狀態可在支援條件下與 OceanAI Cloud 同步。狀態準確性取決於設備連接、上報行為、Runtime 處理與同步狀態。

## 6.7 Command Routing

## 6.7 指令路由

**EN**  
Command routing defines how commands from supported sources are delivered to target components or devices. Commands may originate from Artibird App, Artibird Switch, Artibird Panel, OceanAI Cloud, local automation, scene execution, or OceanAI Edge where supported.

Command routing should follow runtime priority, device availability, permission rules, connectivity status, and validation boundaries.

**ZH-TW**  
指令路由定義來自受支援來源的指令如何被傳遞至目標組件或設備。指令可來自 Artibird App、Artibird Switch、Artibird Panel、OceanAI Cloud、本地自動化、場景執行，或在支援條件下來自 OceanAI Edge。

指令路由應遵循 Runtime 優先級、設備可用性、權限規則、連接狀態與驗證邊界。

## 6.8 Scene Execution

## 6.8 場景執行

**EN**  
Scene execution refers to predefined multi-device behavior executed by the local runtime where supported. A scene may include actions across lighting, plug control, appliance synchronization, room interface display, environmental response, or other supported device behavior.

Scene execution should be controlled by Hub and ArtiOS where required local components are available. Cloud may synchronize scene configuration or trigger remote scene execution where supported.

**ZH-TW**  
場景執行指在支援條件下，由本地 Runtime 執行的預定義多設備行為。場景可包含照明、插座控制、家電同步、房間介面顯示、環境回應或其他受支援設備行為。

在所需本地元件具備時，場景執行應由 Hub 與 ArtiOS 控制。雲端可在支援條件下同步場景配置或觸發遠端場景執行。

---

## 6.9 自動化執行

**EN**  
Automation execution refers to conditional runtime behavior based on time, sensor values, events, state changes, user configuration, cloud input, or edge-assisted interpretation where supported.

Automation should remain within validated device capability, supported local runtime conditions, permission rules, and safety boundaries.

**ZH-TW**  
自動化執行指基於時間、感測值、事件、狀態變化、用戶配置、雲端輸入或支援條件下 Edge 輔助解讀的條件 Runtime 行為。

自動化應保持在已驗證設備能力、受支援本地 Runtime 條件、權限規則與安全邊界內。

## 6.10 Data Synchronization

## 6.10 數據同步

**EN**  
Data synchronization defines how selected state, configuration, logs, lifecycle data, account data, and historical records may synchronize between local runtime services and OceanAI Cloud.

Synchronization behavior must be defined according to data type, network availability, account binding, user permission, cloud availability, and privacy policy.

**ZH-TW**  
數據同步定義特定狀態、配置、日誌、生命週期數據、帳號數據與歷史記錄如何在本地 Runtime 服務與 OceanAI Cloud 之間同步。

同步行為必須根據數據類型、網路可用性、帳號綁定、用戶權限、雲端可用性與隱私政策進行定義。

---

## 6.11 Offline Behavior

## 6.11 離線行為

**EN**  
Offline behavior defines which supported functions can continue when Internet or cloud access is unavailable. Local device control, supported scenes, supported automations, physical controls, and selected local runtime actions may continue where Hub, ArtiOS, Gate, and required devices are available.

Remote access, cloud synchronization, account updates, cloud alerts, multi-site management, and cloud-based historical data may be unavailable or degraded during offline conditions.

**ZH-TW**  
離線行為定義在 Internet 或雲端存取不可用時，哪些受支援功能可以繼續運作。在 Hub、ArtiOS、Gate 與所需設備具備時，本地設備控制、受支援場景、受支援自動化、實體控制與特定本地 Runtime 動作可繼續運作。

在離線條件下，遠端存取、雲端同步、帳號更新、雲端警報、多站點管理與雲端歷史數據可能不可用或降級。

---

## 6.12 Edge Participation

## 6.12 Edge 參與

**EN**  
Edge participation defines how OceanAI Edge may provide AI-assisted interpretation to the local runtime where deployed. Edge may process visual context, sensor data, state data, and selected runtime information to support automation, energy, comfort, security, or contextual interpretation.

Edge-assisted output should be treated as an assisted interpretation unless otherwise validated. It must not bypass safety behavior, user permissions, runtime priority, or defined control boundaries.

**ZH-TW**  
Edge 參與定義 OceanAI Edge 在部署條件下如何向本地 Runtime 提供 AI 輔助解讀。Edge 可處理視覺上下文、感測數據、狀態數據與特定 Runtime 資訊，以支援自動化、能源、舒適度、安全或上下文解讀。

除非另行驗證，Edge 輔助輸出應被視為輔助解讀。它不得繞過安全行為、用戶權限、Runtime 優先級或已定義控制邊界。

## 6.14 Runtime Boundary

## 6.14 Runtime 邊界

**EN**  
Runtime behavior must be described according to supported components, validated device capabilities, firmware versions, software versions, local network conditions, cloud availability, edge deployment status, and configured permissions.

Runtime behavior does not imply universal compatibility, unlimited automation, unrestricted AI control, guaranteed cloud availability, or support for unsupported devices, loads, appliances, or third-party platforms.

**ZH-TW**  
Runtime 行為必須依據受支援組件、已驗證設備能力、韌體版本、軟體版本、本地網路條件、雲端可用性、Edge 部署狀態與已配置權限進行描述。

Runtime 行為不表示通用相容、無限制自動化、不受限制 AI 控制、保證雲端可用，或支援未支援設備、負載、家電或第三方平台。
