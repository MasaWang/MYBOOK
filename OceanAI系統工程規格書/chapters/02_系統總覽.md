# **2. 系統總覽**

## **2.1 Purpose**

## **2.1 目的**

**EN  
**This section defines what the OceanAI / Artibird system is, what
coordinated intelligent environments mean, and how OceanAI, Artibird,
ArtiOS, Cloud, Edge, Hub, Gate, and device nodes operate together.

**ZH-TW  
**本章定義 OceanAI / Artibird 系統是什麼、協同智慧空間是什麼，以及
OceanAI、Artibird、ArtiOS、Cloud、Edge、Hub、Gate
與設備節點如何共同運作。

## **2.2 System Definition**

## **2.2 系統定義**

**EN  
**The OceanAI / Artibird system is a coordinated intelligent environment
architecture that organizes devices, data, sensing, execution, user
interaction, cloud services, and edge intelligence into one runtime
system.

It is not a collection of isolated smart devices. It is a layered system
in which physical devices, local runtime services, cloud coordination,
and edge intelligence participate in shared environmental behavior.

**ZH-TW  
**OceanAI / Artibird
系統是一套協同智慧空間架構，將設備、數據、感知、執行、用戶互動、雲端服務與邊緣智慧組織為一個
Runtime 系統。

它不是一組孤立智慧設備的集合，而是一個分層系統，使實體設備、本地 Runtime
服務、雲端協同與邊緣智慧共同參與共享環境行為。

## **2.3 Architecture Identity**

## **2.3 架構身份**

**EN  
**OceanAI acts as the system-level intelligence architecture. Artibird
acts as the physical product and device layer. ArtiOS provides local
runtime behavior. Artibird Hub hosts local services. Artibird Gate
provides device access. OceanAI Cloud extends remote management. OceanAI
Edge adds local AI-assisted analysis and perception.

**ZH-TW  
**OceanAI 作為系統級智慧架構。Artibird 作為實體產品與設備層。ArtiOS
提供本地 Runtime 行為。Artibird Hub 承載本地服務。Artibird Gate
提供設備接入。OceanAI Cloud 擴展遠端管理。OceanAI Edge 增加本地 AI
輔助分析與感知能力。

## **2.4 Coordinated Intelligent Environment**

## **2.4 協同智慧空間**

**EN  
**A coordinated intelligent environment is a physical space where
supported devices, local runtime services, cloud services, edge
intelligence, sensing nodes, execution nodes, and user interfaces
operate through shared system behavior.

In this model, devices do not only respond as separate endpoints. They
participate in common runtime logic through events, commands, states,
scenes, automation, sensing data, and execution actions.

**ZH-TW  
**協同智慧空間是指一個實體空間中，受支援設備、本地 Runtime
服務、雲端服務、邊緣智慧、感知節點、執行節點與用戶介面透過共享系統行為共同運作。

在此模型中，設備不只是作為分離端點進行回應，而是透過事件、指令、狀態、場景、自動化、感測數據與執行動作參與共同
Runtime 邏輯。

## **2.5 System Participation Model**

## **2.5 系統參與模型**

**EN  
**Each supported component participates in the OceanAI / Artibird system
according to its defined role. Cloud provides remote services. Edge
provides AI-assisted local intelligence. ArtiOS provides runtime logic.
Hub hosts local services. Gate provides device access. Artibird devices
provide interaction, sensing, execution, synchronization, and visual
perception.

**ZH-TW  
**每個受支援組件依其定義角色參與 OceanAI / Artibird 系統。Cloud
提供遠端服務。Edge 提供 AI 輔助本地智慧。ArtiOS 提供 Runtime 邏輯。Hub
承載本地服務。Gate 提供設備接入。Artibird
設備提供互動、感知、執行、同步與視覺感知能力。
