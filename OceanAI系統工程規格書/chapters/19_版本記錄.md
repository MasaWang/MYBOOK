# **2. 前言**

## **EN**

The OceanAI / Artibird System Specification v1.0 defines the
foundational system structure for the OceanAI / Artibird coordinated
intelligent environment system.

This document is not a product catalog, a marketing brochure, or a
single-device specification. It is a system-level reference document
that defines how OceanAI and Artibird components operate together across
architecture, runtime behavior, device participation, data flow,
connectivity, local-first operation, cloud enhancement, edge
intelligence, deployment, validation, and commercial claim boundaries.

OceanAI is defined in this document as the system-level coordinated
intelligence architecture. Artibird is defined as the physical product
and device layer that allows the architecture to take form within real
environments. ArtiOS provides the local runtime structure through which
devices, services, events, commands, scenes, automations, and system
states are processed.

The purpose of this specification is to establish a shared technical
foundation across product engineering, firmware development, backend
services, App design, cloud services, edge intelligence, testing,
certification, deployment planning, commercial documentation, and system
integration.

A coordinated intelligent environment cannot be defined only by
individual products. It must be defined by how components participate
together: how devices are discovered, how events move, how states are
maintained, how commands are routed, how local behavior continues, how
cloud services enhance operation, how edge intelligence assists
interpretation, and how all claims are validated under defined
conditions.

This document therefore serves as the baseline system-level reference
above individual Artibird product engineering specifications. Product
specifications define what each product is. This system specification
defines how the products operate together within the OceanAI / Artibird
system.

All future product documents, commercial specifications, deployment
plans, validation reports, website descriptions, channel materials, and
claim language should remain aligned with the boundaries and validation
principles defined in this document.
