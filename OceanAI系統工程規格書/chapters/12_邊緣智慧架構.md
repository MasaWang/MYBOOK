# **12. 邊緣智慧架構**

## **12.1 Purpose**

## **12.1 目的**

**EN  
**This section defines how OceanAI Edge participates in the system as an
AI-assisted local intelligence layer.

**ZH-TW  
**本章定義 OceanAI Edge 如何作為 AI 輔助本地智慧層參與系統。

## **12.2 Edge Intelligence Architecture Definition**

## **12.2 邊緣智慧架構定義**

**EN  
**Edge Intelligence Architecture defines how OceanAI Edge participates
in the OceanAI / Artibird system through local AI-assisted inference,
visual understanding, sensor fusion, contextual interpretation, and
intelligent optimization where deployed.

OceanAI Edge is intended to enhance local runtime behavior by providing
AI-assisted interpretation. It should not replace deterministic runtime
behavior, safety rules, user permissions, validated control logic, or
defined system boundaries.

**ZH-TW  
**邊緣智慧架構定義 OceanAI Edge 如何在部署條件下，透過本地 AI
輔助推理、視覺理解、感測融合、上下文解讀與智慧優化參與 OceanAI /
Artibird 系統。

OceanAI Edge 用於透過 AI 輔助解讀增強本地 Runtime
行為。它不應取代確定性的 Runtime
行為、安全規則、用戶權限、已驗證控制邏輯或已定義系統邊界。

## **12.3 Edge 責任**

| Sensor fusion | Combine Sense, Vision, state, and execution data |
| AI-assisted optimization | Suggest or inform automation, energy, comfort, or security behavior |

## **12.4 Local AI Inference**

## **12.4 本地 AI 推理**

**EN  
**Local AI inference refers to AI model execution within the local
environment where OceanAI Edge is deployed. It may process supported
visual data, sensor data, state data, device events, or selected runtime
information.

Local AI inference should be used to support interpretation,
classification, recommendation, optimization, and context understanding.
It should not be described as unrestricted autonomous control unless
such behavior is explicitly implemented, validated, and bounded by
safety and permission rules.

**ZH-TW  
**本地 AI 推理指在部署 OceanAI Edge 的本地環境中執行 AI
模型。它可處理受支援視覺數據、感測數據、狀態數據、設備事件或特定 Runtime
資訊。

本地 AI
推理應用於支援解讀、分類、建議、優化與上下文理解。除非相關行為已明確實作、驗證，並受到安全與權限規則約束，否則不應描述為不受限制的自主控制。

## **English Table**

| **Inference Input** | **Possible Edge Output** |
|----|----|
| Vision data | Visual event interpretation, room-state signal, security context |
| Sense data | Environmental context, threshold interpretation, comfort signal |
| Device state | State pattern, abnormal condition, usage context |
| Runtime events | Event classification, recommendation, assisted action |
| Energy data where supported | Usage pattern, optimization suggestion, anomaly signal |
| User-configured context | Scenario-specific recommendation or interpretation |

## **中文表**

| **推理輸入**       | **可能 Edge 輸出**                     |
|--------------------|----------------------------------------|
| Vision 數據        | 視覺事件解讀、房間狀態信號、安全上下文 |
| Sense 數據         | 環境上下文、閾值解讀、舒適度信號       |
| 設備狀態           | 狀態模式、異常條件、使用上下文         |
| Runtime 事件       | 事件分類、建議、輔助動作               |
| 支援條件下能源數據 | 使用模式、優化建議、異常信號           |
| 用戶配置上下文     | 特定場景建議或解讀                     |

## **12.5 SLM Integration**

## **12.5 SLM 整合**

**EN  
**SLM integration refers to the use of small language model-assisted
reasoning within OceanAI Edge where supported. It may help interpret
system context, user intent, device state, scene relationships,
automation logic, diagnostic information, or local knowledge.

SLM-assisted output must be treated as assisted interpretation unless
explicitly validated for direct control. It must follow runtime
priority, user permission, security rules, and defined control
boundaries.

**ZH-TW  
**SLM 整合指在支援條件下，於 OceanAI Edge
中使用小型語言模型輔助推理。它可幫助解讀系統上下文、用戶意圖、設備狀態、場景關係、自動化邏輯、診斷資訊或本地知識。

除非已明確驗證可用於直接控制，SLM 輔助輸出必須被視為輔助解讀。它必須遵循
Runtime 優先級、用戶權限、安全規則與已定義控制邊界。

## **12.6 VLM Integration**

## **12.6 VLM 整合**

**EN  
**VLM integration refers to the use of vision-language model-assisted
understanding within OceanAI Edge where supported. It may help interpret
visual context, room conditions, object presence, activity signals,
security events, or visual-state relationships.

VLM-assisted behavior depends on camera availability, Vision device
capability, model capability, local processing resources, privacy
configuration, and validation results.

**ZH-TW  
**VLM 整合指在支援條件下，於 OceanAI Edge
中使用視覺語言模型輔助理解。它可幫助解讀視覺上下文、房間條件、物體存在、活動信號、安全事件或視覺狀態關係。

VLM 輔助行為取決於攝像設備可用性、Vision
設備能力、模型能力、本地處理資源、隱私配置與驗證結果。

## **English Table**

| **VLM Input**                   | **Possible Use**                         |
|---------------------------------|------------------------------------------|
| Visual scene                    | Room-state interpretation                |
| Motion event                    | Security awareness or automation context |
| Object presence                 | Context signal where supported           |
| Activity pattern                | Assisted event classification            |
| Visual anomaly                  | Alert support or diagnostic context      |
| Combined visual and sensor data | Multi-modal context interpretation       |

## **中文表**

| **VLM 輸入**       | **可能用途**           |
|--------------------|------------------------|
| 視覺場景           | 房間狀態解讀           |
| 移動事件           | 安全感知或自動化上下文 |
| 物體存在           | 支援條件下的上下文信號 |
| 活動模式           | 輔助事件分類           |
| 視覺異常           | 警報支援或診斷上下文   |
| 視覺與感測融合數據 | 多模態上下文解讀       |

## **12.7 Vision Analysis**

## **12.7 視覺分析**

**EN  
**Vision analysis defines how OceanAI Edge may process visual context
from Artibird Vision devices. Vision analysis may support visual event
interpretation, motion analysis, room-state understanding, security
awareness, local alert support, and assisted automation.

Vision analysis must follow privacy requirements, storage boundaries,
user permission, local processing rules, and validation conditions.

**ZH-TW  
**視覺分析定義 OceanAI Edge 如何處理來自 Artibird Vision
設備的視覺上下文。視覺分析可支援視覺事件解讀、移動分析、房間狀態理解、安全感知、本地警報支援與輔助自動化。

視覺分析必須遵循隱私要求、儲存邊界、用戶權限、本地處理規則與驗證條件。

## **12.8 Sensor Fusion**

## **12.8 感測融合**

**EN  
**Sensor fusion defines how OceanAI Edge may combine data from Artibird
Sense, Artibird Vision, device state, execution state, and runtime
context to produce a more complete interpretation of the environment.

Sensor fusion may support comfort optimization, energy-aware behavior,
room-state understanding, security context, or automation suggestions
where supported.

**ZH-TW  
**感測融合定義 OceanAI Edge 如何融合來自 Artibird Sense、Artibird
Vision、設備狀態、執行狀態與 Runtime
上下文的數據，以形成更完整的環境解讀。

感測融合可在支援條件下支援舒適度優化、能源感知行為、房間狀態理解、安全上下文或自動化建議。

## **English Table**

| **Data Source** | **Contribution** |

| Artibird Vision | Visual event, motion, room-state, security context |
| Device State | Current device condition and behavior |
| Execution State | Act, Sync, relay, or appliance action result |
| Runtime Context | Scene, automation, schedule, user action, or system event |
| Cloud Context where supported | Historical or multi-site context where available |

## **中文表**

| **數據來源**         | **貢獻**                               |
|----------------------|----------------------------------------|
| Artibird Sense       | 溫度、濕度、光照、人在、環境數據       |

| 設備狀態             | 當前設備條件與行為                     |

## **12.9 AI-Assisted Optimization**

## **12.9 AI 輔助優化**

**EN  
**AI-assisted optimization refers to the use of Edge interpretation to
suggest or inform supported automation, energy behavior, comfort
control, security response, or operational adjustment.

AI-assisted optimization should not be presented as guaranteed
optimization unless validated under defined conditions. It should be
described as assisted behavior based on supported input data, model
capability, runtime configuration, and validation results.

**ZH-TW  
**AI 輔助優化指利用 Edge
解讀來建議或影響受支援自動化、能源行為、舒適度控制、安全回應或運行調整。

除非已在明確條件下完成驗證，否則不應將 AI
輔助優化描述為保證優化。它應被描述為基於受支援輸入數據、模型能力、Runtime
配置與驗證結果的輔助行為。

## **English Table**

| **Optimization Area** | **Example** |
|----|----|
| Energy | Suggest load reduction or energy-aware automation where supported |
| Comfort | Inform climate, lighting, or scene behavior |
| Security | Assist visual event interpretation or alert prioritization |
| Automation | Suggest rule adjustment or assisted trigger behavior |
| Maintenance | Identify abnormal device patterns or diagnostic signals |
| Space usage | Interpret room usage patterns where supported |

## **中文表**

| **優化領域** | **示例**                                 |
|--------------|------------------------------------------|
| 能源         | 在支援條件下建議負載降低或能源感知自動化 |
| 舒適度       | 影響空調、照明或場景行為                 |
| 安全         | 輔助視覺事件解讀或警報優先級             |
| 自動化       | 建議規則調整或輔助觸發行為               |
| 維護         | 識別異常設備模式或診斷信號               |
| 空間使用     | 在支援條件下解讀房間使用模式             |

## **12.10 Data Boundary**

## **12.10 數據邊界**

**EN  
**Data boundary defines what data may be processed locally by OceanAI
Edge, what data may remain local, what data may synchronize with OceanAI
Cloud, and what data is restricted by privacy policy or user settings.

Edge processing should be described according to data type, user
permission, deployment configuration, storage policy, cloud
synchronization rule, and validation condition.

**ZH-TW  
**數據邊界定義哪些數據可由 OceanAI Edge
本地處理、哪些數據可保留本地、哪些數據可與 OceanAI Cloud
同步，以及哪些數據受到隱私政策或用戶設定限制。

Edge
處理應依據數據類型、用戶權限、部署配置、儲存政策、雲端同步規則與驗證條件進行描述。

## **English Table**

| **Data Type** | **Edge Processing** | **Cloud Sync** |
|----|----|----|
| Sensor data | May be processed locally where supported | May sync where enabled |
| Device state | May be processed locally | May sync where supported |
| Vision event metadata | May be processed locally | May sync where enabled |
| Video / image data | Local processing preferred where configured | Sync only where supported and permitted |
| AI interpretation result | May inform runtime behavior | May sync where enabled |
| Diagnostic data | May support local analysis | May sync where supported |
| User configuration | May guide Edge behavior | May sync with cloud where enabled |

## **中文表**

| **數據類型**    | **Edge 處理**          | **雲端同步**             |
|-----------------|------------------------|--------------------------|
| 感測數據        | 支援條件下可本地處理   | 啟用條件下可同步         |
| 設備狀態        | 可本地處理             | 支援條件下可同步         |
| 視覺事件元數據  | 可本地處理             | 啟用條件下可同步         |
| 視頻 / 圖像數據 | 配置條件下優先本地處理 | 僅在支援且允許條件下同步 |
| AI 解讀結果     | 可影響 Runtime 行為    | 啟用條件下可同步         |
| 診斷數據        | 可支援本地分析         | 支援條件下可同步         |
| 用戶配置        | 可引導 Edge 行為       | 啟用條件下可與雲端同步   |

## **12.11 Privacy Behavior**

## **12.11 隱私行為**

**EN  
**Privacy behavior defines how OceanAI Edge handles privacy-sensitive
data, especially visual data, sensor data, user configuration, and
locally processed AI inputs.

Where configured, privacy-sensitive processing should remain local. Any
synchronization to OceanAI Cloud must follow user permission, data
policy, storage policy, account settings, and applicable privacy
requirements.

**ZH-TW  
**隱私行為定義 OceanAI Edge
如何處理隱私敏感數據，尤其是視覺數據、感測數據、用戶配置與本地處理 AI
輸入。

在配置條件下，隱私敏感處理應保留於本地。任何同步至 OceanAI Cloud
的行為都必須遵循用戶權限、數據政策、儲存政策、帳號設定與適用隱私要求。

## **12.12 Failure Behavior**

## **12.12 故障行為**

**EN  
**Failure behavior defines how the system responds when OceanAI Edge is
unavailable, degraded, overloaded, disconnected, or unable to process
AI-assisted functions.

When OceanAI Edge is unavailable, supported non-edge local runtime
behavior should continue where required local components are present.
AI-assisted interpretation, VLM analysis, SLM reasoning, sensor fusion,
and Edge-assisted optimization may be unavailable or degraded.

**ZH-TW  
**故障行為定義 OceanAI Edge 不可用、降級、過載、斷開或無法處理 AI
輔助功能時，系統如何回應。

當 OceanAI Edge 不可用時，在所需本地元件具備條件下，受支援非 Edge 本地
Runtime 行為應繼續運作。AI 輔助解讀、VLM 分析、SLM 推理、感測融合與 Edge
輔助優化可能不可用或降級。

## **English Table**

| **Edge Condition** | **Expected Behavior** |
|----|----|
| Edge available | AI-assisted interpretation may support runtime behavior |
| Edge delayed | AI-assisted output may be delayed or ignored depending on runtime configuration |
| Edge unavailable | Non-edge local runtime behavior continues where supported |
| Edge overloaded | AI-assisted functions may degrade or stop |
| Edge disconnected | Hub / ArtiOS continue supported local runtime behavior |
| Model unavailable | Related AI-assisted function is unavailable |
| Privacy restriction enabled | Restricted data is not processed or not shared according to policy |

## **中文表**

| **Edge 狀態**  | **預期行為**                             |
|----------------|------------------------------------------|
| Edge 可用      | AI 輔助解讀可支援 Runtime 行為           |
| Edge 延遲      | AI 輔助輸出可依 Runtime 配置延遲或被忽略 |
| Edge 不可用    | 支援條件下非 Edge 本地 Runtime 行為繼續  |
| Edge 過載      | AI 輔助功能可能降級或停止                |
| Edge 斷開      | Hub / ArtiOS 繼續受支援本地 Runtime 行為 |
| 模型不可用     | 相關 AI 輔助功能不可用                   |
| 已啟用隱私限制 | 受限制數據依政策不被處理或不被共享       |

## **12.13 Edge Boundary**

## **12.13 Edge 邊界**

**EN  
**Edge Intelligence Architecture must be described according to deployed
hardware, supported model type, software version, device capability,
input data type, privacy configuration, local processing capacity,
runtime integration, and validation evidence.

Edge participation does not imply fully autonomous AI control,
guaranteed AI accuracy, unlimited model capability, universal visual
understanding, or support for all sensor and vision scenarios.

**ZH-TW  
**邊緣智慧架構必須依部署硬體、受支援模型類型、軟體版本、設備能力、輸入數據類型、隱私配置、本地處理能力、Runtime
整合與驗證證據進行描述。

Edge 參與不表示完全自主 AI 控制、不表示保證 AI
準確性、不表示無限制模型能力、不表示通用視覺理解，也不表示支援所有感測與視覺場景。

## **12.14 Validation Requirements**

## **12.14 驗證要求**

**EN  
**Any Edge-related claim involving AI inference, SLM reasoning, VLM
understanding, vision analysis, sensor fusion, optimization, privacy
behavior, latency, reliability, or failure behavior must be validated
under defined conditions.

Validation should include hardware platform, model version, software
version, input data type, processing load, local network condition,
runtime integration path, expected output, failure behavior, and privacy
configuration where applicable.

**ZH-TW  
**任何涉及 Edge 的宣稱，包括 AI 推理、SLM 推理、VLM
理解、視覺分析、感測融合、優化、隱私行為、延遲、可靠性或故障行為，都必須在明確條件下完成驗證。

驗證應包含硬體平台、模型版本、軟體版本、輸入數據類型、處理負載、本地網路條件、Runtime
整合路徑、預期輸出、故障行為，以及適用時的隱私配置。
