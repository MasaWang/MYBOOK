# 2. Preface

# 2. 前言

## EN

The OceanAI / Artibird System Specification v1.0 defines the foundational system structure for the OceanAI / Artibird coordinated intelligent environment system.

This document is not a product catalog, a marketing brochure, or a single-device specification. It is a system-level reference document that defines how OceanAI and Artibird components operate together across architecture, runtime behavior, device participation, data flow, connectivity, local-first operation, cloud enhancement, edge intelligence, deployment, validation, and commercial claim boundaries.

OceanAI is defined in this document as the system-level coordinated intelligence architecture. Artibird is defined as the physical product and device layer that allows the architecture to take form within real environments. ArtiOS provides the local runtime structure through which devices, services, events, commands, scenes, automations, and system states are processed.

The purpose of this specification is to establish a shared technical foundation across product engineering, firmware development, backend services, App design, cloud services, edge intelligence, testing, certification, deployment planning, commercial documentation, and system integration.

A coordinated intelligent environment cannot be defined only by individual products. It must be defined by how components participate together: how devices are discovered, how events move, how states are maintained, how commands are routed, how local behavior continues, how cloud services enhance operation, how edge intelligence assists interpretation, and how all claims are validated under defined conditions.

This document therefore serves as the baseline system-level reference above individual Artibird product engineering specifications. Product specifications define what each product is. This system specification defines how the products operate together within the OceanAI / Artibird system.

All future product documents, commercial specifications, deployment plans, validation reports, website descriptions, channel materials, and claim language should remain aligned with the boundaries and validation principles defined in this document.

## ZH-TW

《OceanAI / Artibird 系統規格書 v1.0》定義 OceanAI / Artibird 協同智慧空間系統的基礎系統結構。

本文件不是產品型錄、行銷手冊，也不是單一設備規格書。它是一份系統級參考文件，用於定義 OceanAI 與 Artibird 組件如何在系統架構、Runtime 行為、設備參與、數據流、連接方式、本地優先運作、雲端增強、邊緣智慧、部署、驗證與商業宣稱邊界中共同運作。

在本文件中，OceanAI 被定義為系統級協同智慧架構。Artibird 被定義為使該架構能夠在真實環境中成形的實體產品與設備層。ArtiOS 則提供本地 Runtime 結構，用於處理設備、服務、事件、指令、場景、自動化與系統狀態。

本規格書的目的，是在產品工程、韌體開發、後端服務、App 設計、雲端服務、邊緣智慧、測試、認證、部署規劃、商業文件與系統整合之間，建立共同的技術基準。

一個協同智慧空間不能只由單一產品來定義，而必須由各組件如何共同參與來定義：設備如何被發現，事件如何流動，狀態如何維護，指令如何路由，本地行為如何延續，雲端服務如何增強運作，邊緣智慧如何輔助解讀，以及所有宣稱如何在明確條件下完成驗證。

因此，本文件作為高於各個 Artibird 單品工程規格書之上的基礎系統級參考文件。產品規格書定義每個產品是什麼；本系統規格書定義這些產品如何在 OceanAI / Artibird 系統中共同運作。

未來所有產品文件、商業規格書、部署方案、驗證報告、網站描述、渠道材料與宣稱語言，都應與本文件所定義的邊界與驗證原則保持一致。
