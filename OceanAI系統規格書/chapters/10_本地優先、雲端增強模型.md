# **10. 本地優先、雲端增強模型**

## **10.1 Purpose**

## **10.1 目的**

**EN  
**This section defines what works locally, what requires cloud, and what
is enhanced by cloud.

**ZH-TW  
**本章定義哪些功能可本地運作、哪些需要雲端，以及哪些由雲端增強。

## **10.2 Local-First, Cloud-Enhanced Definition**

## **10.2 本地優先、雲端增強定義**

**EN  
**The Local-First, Cloud-Enhanced Model defines the operating
relationship between local runtime services and OceanAI Cloud services.

In this model, core local behavior should be maintained by Artibird Hub,
ArtiOS, Artibird Gate, and supported local devices where required
components are present. OceanAI Cloud extends the system through remote
access, synchronization, account services, alerts, lifecycle management,
historical data, analytics, and multi-site management where supported.

**ZH-TW  
**本地優先、雲端增強模型定義本地 Runtime 服務與 OceanAI Cloud
服務之間的運作關係。

在此模型中，當所需元件具備時，核心本地行為應由 Artibird
Hub、ArtiOS、Artibird Gate 與受支援本地設備維持。OceanAI Cloud
則在支援條件下透過遠端存取、同步、帳號服務、警報、生命週期管理、歷史數據、分析與多站點管理擴展系統能力。

## **10.3 Operating Principle**

## **10.3 運作原則**

**EN  
**The system should avoid making every essential operation dependent on
cloud availability. Local device control, supported scenes, and
supported automations should operate locally when Hub, Gate, ArtiOS, and
required devices are available.

Cloud services should enhance the system but should not be treated as
the mandatory execution path for all local runtime behavior.

**ZH-TW  
**系統應避免讓所有關鍵操作完全依賴雲端可用性。在 Hub、Gate、ArtiOS
與所需設備具備時，本地設備控制、受支援場景與受支援自動化應可本地運作。

雲端服務應增強系統能力，但不應被視為所有本地 Runtime
行為的必要執行路徑。

## **10.5 Local Device Control**

## **10.5 本地設備控制**

**EN  
**Local device control refers to supported device control within the
local environment without requiring cloud execution. This may include
switching, relay control, plug control, scene execution, panel
operation, local App control, and selected automation behavior.

Local device control depends on Hub availability, ArtiOS runtime
services, device connectivity, Gate availability where applicable, user
permission, and supported product capability.

**ZH-TW  
**本地設備控制指在不需要雲端執行的情況下，於本地環境內進行受支援設備控制。這可包括開關控制、繼電器控制、插座控制、場景執行、面板操作、App
本地控制與特定自動化行為。

本地設備控制取決於 Hub 可用性、ArtiOS Runtime 服務、設備連接、適用時
Gate 可用性、用戶權限與受支援產品能力。

## **English Table**

| **Local Function** | **Required Local Components** |

| Local panel control | Panel, Hub / ArtiOS, target device |
| Local App control | App, local network, Hub / ArtiOS, target device |
| Local scene execution | Hub / ArtiOS, configured scene, target devices |
| Local automation | Hub / ArtiOS, rule configuration, required input device, target device |
| Local status update | Device, Hub / ArtiOS, local communication path |

## **10.6 Scene Execution**

## **10.6 場景執行**

**EN  
**Scene execution may operate locally where scenes are configured in the
local runtime and required devices are available. A scene may include
lighting control, plug control, relay control, IR / RF appliance
synchronization, panel display behavior, or other supported device
actions.

Cloud may enhance scene execution by synchronizing scene configuration,
allowing remote scene control, or supporting cross-site scene management
where supported.

**ZH-TW  
**在場景已配置於本地 Runtime
且所需設備可用時，場景執行可在本地運作。場景可包含照明控制、插座控制、繼電器控制、IR
/ RF 家電同步、面板顯示行為或其他受支援設備動作。

雲端可在支援條件下透過同步場景配置、允許遠端場景控制或支援跨站點場景管理來增強場景執行。

## **10.7 Automation**

## **10.7 自動化**

**EN  
**Automation may operate locally where automation rules, input devices,
target devices, and local runtime services are available. Local
automation may be triggered by time, device events, sensor values, state
changes, user actions, or supported runtime conditions.

Cloud may enhance automation by synchronizing rule configuration,
enabling remote management, storing history, delivering alerts, or
supporting multi-site visibility where supported.

**ZH-TW  
**當自動化規則、輸入設備、目標設備與本地 Runtime
服務可用時，自動化可在本地運作。本地自動化可由時間、設備事件、感測值、狀態變化、用戶動作或受支援
Runtime 條件觸發。

雲端可在支援條件下透過同步規則配置、啟用遠端管理、儲存歷史、推送警報或支援多站點可視化來增強自動化。

## **English Table**

| **Automation Type** | **Local Availability** | **Cloud Enhancement** |
|----|----|----|
| Time-based automation | Available where local schedule is supported | Remote schedule management where supported |
| Sensor-based automation | Available with local Sense / input device and runtime rules | Historical records and remote alerts where supported |
| Scene-based automation | Available with local scene configuration | Cloud synchronization of scene configuration |
| Device-state automation | Available with local state handling | Multi-site state visibility where supported |
| Vision-based automation | Partial / where supported | Cloud history or Edge-assisted enhancement where supported |
| Energy-aware automation | Partial / where supported | Reports, analytics, and remote monitoring where supported |

## **中文表**

| **自動化類型** | **本地可用性** | **雲端增強** |
|----|----|----|
| 基於時間的自動化 | 支援本地排程時可用 | 支援條件下遠端排程管理 |
| 基於感測的自動化 | 本地 Sense / 輸入設備與 Runtime 規則可用時可用 | 支援條件下歷史記錄與遠端警報 |
| 基於場景的自動化 | 本地場景配置可用時可用 | 場景配置雲端同步 |
| 基於設備狀態的自動化 | 本地狀態處理可用時可用 | 支援條件下多站點狀態可視化 |
| 基於 Vision 的自動化 | 部分 / 支援條件下可用 | 支援條件下雲端歷史或 Edge 輔助增強 |
| 能源感知自動化 | 部分 / 支援條件下可用 | 支援條件下報告、分析與遠端監測 |

## **10.8 Remote Control**

## **10.8 遠端控制**

**EN  
**Remote control requires cloud participation. It allows users to
control supported devices, scenes, automations, or sites through
Artibird App or supported remote interfaces when Internet access,
account binding, permissions, Hub availability, and cloud service
availability are present.

Remote control does not replace local control. It extends user access
beyond the local environment where supported.

**ZH-TW  
**遠端控制需要雲端參與。當 Internet 存取、帳號綁定、權限、Hub
可用性與雲端服務可用時，用戶可透過 Artibird App
或受支援遠端介面控制受支援設備、場景、自動化或站點。

遠端控制不取代本地控制，而是在支援條件下將用戶存取能力擴展至本地環境之外。

## **10.9 Multi-Site Management**

## **10.9 多站點管理**

**EN  
**Multi-site management is primarily a cloud-enhanced capability. It may
support multiple homes, classrooms, offices, buildings, campuses, or
infrastructure sites under one account or organization structure where
supported.

Local runtime services may manage local behavior within each site, while
OceanAI Cloud may provide cross-site visibility, account management,
reporting, alerts, synchronization, and lifecycle services.

**ZH-TW  
**多站點管理主要是雲端增強能力。它可在支援條件下，在同一帳號或組織結構下管理多住宅、教室、辦公室、建築、校園或基礎設施站點。

本地 Runtime 服務可管理每個站點內的本地行為，而 OceanAI Cloud
可提供跨站點可視化、帳號管理、報告、警報、同步與生命週期服務。

## **10.10 Account Binding**

## **10.10 帳號綁定**

**EN  
**Account binding associates users, devices, rooms, sites, permissions,
and cloud services. Some local setup or local device discovery may be
possible without full cloud dependency depending on implementation, but
account-based remote access, multi-site management, and cloud
synchronization generally require cloud participation.

Account binding must follow security, privacy, and permission rules.

**ZH-TW  
**帳號綁定將用戶、設備、房間、站點、權限與雲端服務關聯起來。依實作方式不同，部分本地設定或本地設備發現可不完全依賴雲端，但基於帳號的遠端存取、多站點管理與雲端同步通常需要雲端參與。

帳號綁定必須遵循安全、隱私與權限規則。

## **10.11 Firmware Update**

## **10.11 韌體更新**

**EN  
**Firmware update may be local, cloud-enhanced, or cloud-required
depending on product type and implementation. Cloud services may provide
firmware package delivery, update availability, version management,
update records, and lifecycle coordination where supported.

Firmware update behavior must define update source, device eligibility,
version compatibility, failure behavior, rollback behavior where
supported, and validation requirements.

**ZH-TW  
**依產品類型與實作方式不同，韌體更新可為本地、雲端增強或需要雲端。雲端服務可在支援條件下提供韌體包分發、更新可用性、版本管理、更新記錄與生命週期協調。

韌體更新行為必須定義更新來源、設備資格、版本相容性、失敗行為、支援條件下回滾行為與驗證要求。

## **10.12 Historical Analytics**

## **10.12 歷史分析**

**EN  
**Historical analytics is primarily cloud-enhanced. The local runtime
may maintain limited state, logs, and recent records, while OceanAI
Cloud may provide long-term history, trends, reports, energy analysis,
device usage records, and multi-site analytics where supported.

Historical analytics must follow storage, privacy, retention, and user
permission rules.

**ZH-TW  
**歷史分析主要由雲端增強。本地 Runtime
可維護有限狀態、日誌與近期記錄，而 OceanAI Cloud
可在支援條件下提供長期歷史、趨勢、報告、能源分析、設備使用記錄與多站點分析。

歷史分析必須遵循儲存、隱私、保留與用戶權限規則。

## **10.13 Alerts**

## **10.13 警報**

**EN  
**Alerts may be generated locally, cloud-enhanced, or cloud-delivered
depending on event type and deployment configuration. Local alerts may
be shown through Panel, App local access, device indicators, or local
runtime logs where supported.

Cloud-enhanced alerts may include push notifications, remote event
delivery, multi-site alerts, historical alert records, and account-based
notification rules where supported.

**ZH-TW  
**依事件類型與部署配置不同，警報可由本地生成、雲端增強或透過雲端傳送。本地警報可在支援條件下透過
Panel、App 本地存取、設備指示燈或本地 Runtime 日誌呈現。

雲端增強警報可在支援條件下包含推送通知、遠端事件傳送、多站點警報、歷史警報記錄與基於帳號的通知規則。

## **10.14 Lifecycle Management**

## **10.14 生命週期管理**

**EN  
**Lifecycle management includes device onboarding, binding,
configuration, firmware update, diagnostics, support records,
replacement, reset, decommissioning, and service history.

Local runtime may support selected lifecycle actions, while OceanAI
Cloud may enhance lifecycle management through account-based records,
update coordination, service history, diagnostics, and multi-site device
management where supported.

**ZH-TW  
**生命週期管理包括設備入網、綁定、配置、韌體更新、診斷、支援記錄、更換、重置、退役與服務歷史。

本地 Runtime 可支援特定生命週期動作，而 OceanAI Cloud
可在支援條件下透過基於帳號的記錄、更新協調、服務歷史、診斷與多站點設備管理增強生命週期管理。

## **10.15 Cloud Dependency Boundary**

## **10.15 雲端依賴邊界**

**EN  
**Cloud dependency must be clearly defined for each system capability. A
capability should not be described as fully local if it requires cloud
access. A capability should not be described as cloud-required if its
core behavior can operate locally with required local components.

Cloud dependency should be evaluated according to function type, device
type, account requirement, synchronization need, remote access
requirement, data storage requirement, and validation results.

**ZH-TW  
**每一項系統能力都必須清楚定義雲端依賴程度。如果某項能力需要雲端存取，不應描述為完全本地。如果某項能力在所需本地元件具備時可本地運作，則不應描述為必須依賴雲端。

雲端依賴應依功能類型、設備類型、帳號要求、同步需求、遠端存取需求、數據儲存需求與驗證結果進行評估。

## **English Table**

| **Dependency Type** | **Description** |
|----|----|
| Local-only | Function operates without cloud where required local components are present |
| Local with cloud enhancement | Function operates locally and may be improved by cloud services |
| Cloud-required | Function requires cloud service availability |
| Cloud-degraded | Function partially operates but loses cloud-based capability |
| Cloud-unavailable | Function cannot operate without cloud |
| Hybrid | Function uses both local runtime and cloud services depending on configuration |

## **中文表**

| **依賴類型**       | **說明**                                   |
|--------------------|--------------------------------------------|
| 僅本地             | 所需本地元件具備時，功能不需要雲端即可運作 |
| 本地 + 雲端增強    | 功能可本地運作，並可由雲端服務增強         |
| 必須雲端           | 功能需要雲端服務可用                       |
| 雲端降級           | 功能可部分運作，但失去雲端能力             |
| 雲端不可用即不可用 | 無雲端時功能不可運作                       |
| 混合               | 功能依配置同時使用本地 Runtime 與雲端服務  |

## **10.16 Model Boundary**

## **10.16 模型邊界**

**EN  
**The Local-First, Cloud-Enhanced Model does not imply that every
function is fully local, nor does it imply that every function requires
cloud. Each capability must be classified according to supported
implementation, product type, deployment model, network condition, cloud
availability, user permission, and validation evidence.

This model must not be used to claim unlimited offline operation,
universal remote access, guaranteed cloud availability, or support for
unsupported devices and third-party platforms.

**ZH-TW  
**本地優先、雲端增強模型不表示所有功能都完全本地，也不表示所有功能都需要雲端。每一項能力都必須依受支援實作、產品類型、部署模型、網路條件、雲端可用性、用戶權限與驗證證據進行分類。

不得使用此模型宣稱無限制離線運作、通用遠端存取、保證雲端可用，或支援未支援設備與第三方平台。

## **10.17 Validation Requirements**

## **10.17 驗證要求**

**EN  
**Any claim involving local operation, offline behavior, remote control,
cloud synchronization, cloud-enhanced analytics, firmware update,
alerts, lifecycle management, or multi-site management must be validated
under defined conditions.

Validation should include required local components, cloud availability,
network condition, user permissions, device count, firmware version,
software version, failure behavior, and expected recovery behavior where
applicable.

**ZH-TW  
**任何涉及本地運作、離線行為、遠端控制、雲端同步、雲端增強分析、韌體更新、警報、生命週期管理或多站點管理的宣稱，都必須在明確條件下完成驗證。

驗證應包含所需本地組件、雲端可用性、網路條件、用戶權限、設備數量、韌體版本、軟體版本、故障行為，以及適用時的預期恢復行為。
