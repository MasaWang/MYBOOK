## **15.2 Deployment Model Definition**

## **15.2 部署模型定義**

**EN  
**Deployment Models define how the OceanAI / Artibird system is
organized and installed within supported physical environments. A
deployment model describes the required components, system role, local
runtime structure, cloud participation, edge participation, device
categories, network requirements, and validation boundaries for a
specific scenario.

A deployment model does not imply that every component is required in
every scenario. Each deployment should be defined according to the
actual environment, required functions, supported products, network
condition, local runtime requirement, cloud requirement, edge
requirement, and validation evidence.

**ZH-TW  
**部署模型定義 OceanAI / Artibird
系統如何在受支援實體環境中組織與安裝。部署模型描述特定場景所需組件、系統角色、本地
Runtime 結構、雲端參與、Edge 參與、設備類別、網路要求與驗證邊界。

部署模型不表示每個場景都必須使用所有組件。每個部署都應依實際環境、所需功能、受支援產品、網路條件、本地
Runtime 要求、雲端要求、Edge 要求與驗證證據進行定義。

## **15.7 Smart Building**

## **15.7 智慧建築**

**EN

environmental sensing, room-level interface, gateway access, local

awareness.

This deployment model requires clear zoning, component placement,
network planning, capacity planning, local runtime planning, and
validation of device count, message flow, and failure behavior.

**ZH-TW  
**智慧建築指建築內多房間或多區域部署方式。它可包含照明控制、電力控制、環境感知、房間級介面、網關接入、本地
Runtime 協同、雲端增強監測與特定視覺感知。

此部署模型需要明確分區、組件佈置、網路規劃、容量規劃、本地 Runtime
規劃，以及設備數量、消息流與故障行為驗證。

## **15.13 部署邊界**

**EN  
**Deployment models must be described according to actual site
conditions, supported products, validated network conditions, power
conditions, device placement, environmental conditions, cloud
availability, edge deployment status, and system capacity limits.

A deployment model does not imply universal compatibility, unlimited
device count, guaranteed network coverage, guaranteed energy savings,
perfect AI vision detection, or support for all building, campus,
agriculture, or infrastructure environments.

**ZH-TW  
**部署模型必須依實際場地條件、受支援產品、已驗證網路條件、供電條件、設備佈置、環境條件、雲端可用性、Edge
部署狀態與系統容量限制進行描述。

部署模型不表示通用相容、不表示無限制設備數量、不表示保證網路覆蓋、不表示保證節能、不表示完美
AI 視覺偵測，也不表示支援所有建築、校園、農業或基礎設施環境。

## **15.14 Validation Requirements**

## **15.14 驗證要求**

**EN  
**Any deployment claim involving supported scenario, device count,
network coverage, local operation, cloud-enhanced management,
edge-assisted behavior, energy management, vision-aware security,
outdoor use, or multi-site management must be validated under defined
deployment conditions.

Validation should include site type, product type, hardware version,
firmware version, software version, device count, network topology,
power condition, cloud condition, Edge condition, environmental
condition, expected behavior, failure behavior, and observed result.

**ZH-TW  
**任何涉及受支援場景、設備數量、網路覆蓋、本地運作、雲端增強管理、Edge
輔助行為、能源管理、視覺感知安全、戶外使用或多站點管理的部署宣稱，都必須在明確部署條件下完成驗證。

驗證應包含場地類型、產品類型、硬體版本、韌體版本、軟體版本、設備數量、網路拓撲、供電條件、雲端條件、Edge
條件、環境條件、預期行為、故障行為與觀察結果。
