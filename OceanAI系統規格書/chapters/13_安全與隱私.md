## **13.2 Security and Privacy Definition**

## **13.2 安全與隱私定義**

**EN  
**Security and Privacy define how the OceanAI / Artibird system protects
supported devices, user accounts, local runtime services, cloud
services, edge intelligence, data flow, remote access, and system
control behavior.

Security is responsible for device identity, user authentication, access
control, communication protection, firmware update protection, and
system operation integrity.

Privacy is responsible for defining how user data, device data, sensor
data, visual data, cloud data, edge-processed data, logs, and
account-related information are collected, stored, processed,
synchronized, retained, and accessed.

**ZH-TW  
**安全與隱私定義 OceanAI / Artibird
系統如何保護受支援設備、用戶帳號、本地 Runtime
服務、雲端服務、邊緣智慧、數據流、遠端存取與系統控制行為。

安全負責設備身份、用戶身份認證、存取控制、通訊保護、韌體更新保護與系統運作完整性。

隱私負責定義用戶數據、設備數據、感測數據、視覺數據、雲端數據、Edge
處理數據、日誌與帳號相關資訊如何被收集、儲存、處理、同步、保留與存取。

## **13.4 Device Identity**

## **13.4 設備身份**

**EN  
**Device identity defines how each supported device is uniquely
identified and authenticated within the OceanAI / Artibird system.
Device identity may include Device ID, product type, model number,
hardware version, firmware version, security credential, account binding
status, and trusted registration status.

Device identity is required for device onboarding, access control,
command routing, cloud synchronization, firmware update, security
logging, and system validation.

**ZH-TW

ID、產品類型、型號、硬體版本、韌體版本、安全憑證、帳號綁定狀態與可信註冊狀態。

設備身份對於設備入網、存取控制、指令路由、雲端同步、韌體更新、安全日誌與系統驗證都是必要的。

## **13.5 Account Binding**

## **13.5 帳號綁定**

**EN  
**Account binding defines the secure association between a supported
device and a user account, home, site, organization, room, or system
deployment.

Account binding must verify device identity, user permission, ownership
status, site assignment, and security status where supported. Binding
behavior should prevent unauthorized device takeover, invalid
re-binding, unsafe migration, and unsupported system participation.

**ZH-TW  
**帳號綁定定義受支援設備與用戶帳號、住宅、站點、組織、房間或系統部署之間的安全關聯。

帳號綁定必須在支援條件下驗證設備身份、用戶權限、所有權狀態、站點分配與安全狀態。綁定行為應防止未授權設備接管、無效重新綁定、不安全遷移與未支援系統參與。

implementation specifications.

權限、消息驗證、保留消息策略與故障處理。

results, Vision metadata, and account-related information.

**ZH-TW

Audit logs may be used for diagnostics, security review, validation,

審計日誌可在適用條件下用於診斷、安全審查、驗證、帳號管理、故障排查與合規相關審查。

## **13.16 Security and Privacy Boundary**

## **13.16 安全與隱私邊界**

**EN  
**Security and privacy behavior must be described according to
implemented features, supported devices, software version, firmware
version, account configuration, local network condition, cloud
availability, Edge deployment status, privacy settings, and validation
evidence.

Security and privacy definitions do not imply absolute security,
unlimited privacy protection, unrestricted remote access, unsupported
encryption, unsupported local AI privacy behavior, or compliance with
every external regulatory requirement unless explicitly verified.

**ZH-TW  
**安全與隱私行為必須依已實作功能、受支援設備、軟體版本、韌體版本、帳號配置、本地網路條件、雲端可用性、Edge
部署狀態、隱私設定與驗證證據進行描述。

安全與隱私定義不表示絕對安全、不表示無限制隱私保護、不表示不受限制遠端存取、不表示支援未實作加密、不表示支援未定義本地
AI 隱私行為，也不表示符合所有外部法規要求，除非已明確驗證。

## **13.17 Validation Requirements**

## **13.17 驗證要求**

**EN  
**Any security or privacy claim involving device identity, account
binding, access control, secure MQTT, firmware update security, data
retention, Vision privacy, cloud data policy, Edge processing boundary,
role-based access control, or audit logs must be validated under defined
conditions.

Validation should include product type, firmware version, software
version, account role, access path, network condition, cloud
availability, local runtime availability, Edge deployment status,
expected behavior, failure behavior, and privacy configuration where
applicable.

**ZH-TW  
**任何涉及設備身份、帳號綁定、存取控制、安全
MQTT、韌體更新安全、數據保留、Vision 隱私、雲端數據政策、Edge
處理邊界、角色權限控制或審計日誌的安全或隱私宣稱，都必須在明確條件下完成驗證。

驗證應包含產品類型、韌體版本、軟體版本、帳號角色、存取路徑、網路條件、雲端可用性、本地
Runtime 可用性、Edge
部署狀態、預期行為、故障行為，以及適用時的隱私配置。
