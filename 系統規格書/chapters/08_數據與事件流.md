# **Chapter 8 數據與事件流**

# **8. Data and Event Flow**

# **8. 數據與事件流**

## **8.1 Purpose**

## **8.1 目的**

**EN  
**This section defines how events, state, commands, telemetry, and
synchronization data move through the system.

**ZH-TW  
**本章定義事件、狀態、指令、遙測數據與同步數據如何在系統中流動。

## **8.2 Data and Event Flow Definition**

## **8.2 數據與事件流定義**

**EN  
**Data and event flow refers to the movement of system information
between supported devices, Artibird Gate, Artibird Hub, ArtiOS, OceanAI
Cloud, OceanAI Edge, Artibird App, and other supported components.

The system may process device events, user commands, sensor readings,
visual events, execution responses, synchronization data, configuration
updates, lifecycle records, and runtime state changes. These flows allow
supported components to participate in one coordinated intelligent
environment system.

**ZH-TW  
**數據與事件流指系統資訊在受支援設備、Artibird Gate、Artibird
Hub、ArtiOS、OceanAI Cloud、OceanAI Edge、Artibird App
與其他受支援組件之間的流動。

系統可處理設備事件、用戶指令、感測讀數、視覺事件、執行回應、同步數據、配置更新、生命週期記錄與
Runtime 狀態變化。這些流程使受支援組件能夠參與同一個協同智慧空間系統。

## **8.3 主要流程表**

## **English Table**

| Sensor Automation Flow | Sense → Hub → ArtiOS → Act / Sync / Switch | Sensor data triggers local automation |
| Vision Event Flow | Vision → Hub / Edge → ArtiOS / Cloud | Visual events support alerts or automation |
| Cloud Command Flow | Cloud → Hub → Gate / Device | Remote command enters local system |
| Edge Intelligence Flow | Sense / Vision / State → Edge → Hub / ArtiOS | Edge-assisted interpretation informs local behavior |

## **中文表**

|----|----|----|

| 用戶控制流 | App / Panel / Switch → Hub → Device | 用戶動作觸發設備行為 |
| 感測自動化流 | Sense → Hub → ArtiOS → Act / Sync / Switch | 感測數據觸發本地自動化 |
| 視覺事件流 | Vision → Hub / Edge → ArtiOS / Cloud | 視覺事件支援警報或自動化 |
| 雲端指令流 | Cloud → Hub → Gate / Device | 遠端指令進入本地系統 |
| 邊緣智慧流 | Sense / Vision / State → Edge → Hub / ArtiOS | Edge 輔助解讀影響本地行為 |

## **8.4 Device Event Flow**

## **8.4 設備事件流**

**EN  
**Device Event Flow defines how events generated by supported Artibird
devices enter the local runtime system. A device may generate touch
events, relay state changes, sensor readings, IR / RF command results,
connection status changes, visual alerts, or execution responses.

Where applicable, device events may pass through Artibird Gate before
reaching Artibird Hub and ArtiOS. Once received by the local runtime,
the event may update device state, trigger automation, start a scene,
generate an alert, or synchronize with OceanAI Cloud.

**ZH-TW  
**設備事件流定義受支援 Artibird 設備產生的事件如何進入本地 Runtime
系統。設備可產生觸控事件、繼電器狀態變化、感測讀數、IR / RF
指令結果、連接狀態變化、視覺警報或執行回應。

在適用條件下，設備事件可先經由 Artibird Gate，再到達 Artibird Hub 與
ArtiOS。事件進入本地 Runtime
後，可更新設備狀態、觸發自動化、啟動場景、生成警報或與 OceanAI Cloud
同步。

## **8.5 User Control Flow**

## **8.5 用戶控制流**

**EN  
**User Control Flow defines how user actions from Artibird App, Artibird
Panel, or Artibird Switch are converted into system commands or runtime
events.

User control may include local device control, scene selection,
room-level operation, relay control, automation configuration, or remote
control where cloud access is supported. User control behavior must
follow device availability, permission rules, runtime priority,
connectivity status, and validation boundaries.

**ZH-TW  
**用戶控制流定義來自 Artibird App、Artibird Panel 或 Artibird Switch
的用戶動作如何被轉化為系統指令或 Runtime 事件。

用戶控制可包括本地設備控制、場景選擇、房間級操作、繼電器控制、自動化配置，或在支援雲端存取條件下的遠端控制。用戶控制行為必須遵循設備可用性、權限規則、Runtime
優先級、連接狀態與驗證邊界。

## **English Table**

| **User Source** | **Flow Path** | **Possible Result** |
|----|----|----|
| Artibird App | App → Hub / Cloud → Device | Local or remote command execution |
| Artibird Panel | Panel → Hub / ArtiOS → Device Group / Scene | Room-level scene or device control |
| Artibird Switch | Switch → Hub / ArtiOS → Act / Sync / Scene | Touch event, relay control, or scene trigger |
| Cloud Remote User | Cloud → Hub → Gate / Device | Remote control where supported |
| Local User Action | Local interface → Hub / ArtiOS | Local runtime command or state change |

## **中文表**

| **用戶來源** | **流程路徑** | **可能結果** |
|----|----|----|
| Artibird App | App → Hub / Cloud → Device | 本地或遠端指令執行 |
| Artibird Panel | Panel → Hub / ArtiOS → Device Group / Scene | 房間級場景或設備控制 |
| Artibird Switch | Switch → Hub / ArtiOS → Act / Sync / Scene | 觸控事件、繼電器控制或場景觸發 |
| 雲端遠端用戶 | Cloud → Hub → Gate / Device | 支援條件下遠端控制 |
| 本地用戶動作 | 本地介面 → Hub / ArtiOS | 本地 Runtime 指令或狀態變化 |

## **8.6 Sensor Automation Flow**

## **8.6 感測自動化流**

**EN  
**Sensor Automation Flow defines how environmental data from Artibird
Sense or supported sensing devices may trigger local automation. Sensor
data may include temperature, humidity, light, occupancy, environmental
thresholds, or other supported sensor readings.

Sensor automation behavior should be processed by Hub and ArtiOS where
required local components are available. It may result in Act execution,
Sync appliance control, Switch relay behavior, Panel status update,
scene execution, alert generation, or cloud synchronization where
supported.

**ZH-TW  
**感測自動化流定義來自 Artibird Sense
或受支援感測設備的環境數據如何觸發本地自動化。感測數據可包括溫度、濕度、光照、人在、環境閾值或其他受支援感測讀數。

在所需本地元件具備時，感測自動化行為應由 Hub 與 ArtiOS
處理。它可在支援條件下產生 Act 執行、Sync 家電控制、Switch
繼電器行為、Panel 狀態更新、場景執行、警報生成或雲端同步。

## **English Table**

| **Sensor Input** | **Runtime Processing** | **Possible Output** |
|----|----|----|
| Temperature | Hub / ArtiOS rule processing | Sync AC command, Act control, alert |
| Humidity | Hub / ArtiOS rule processing | Ventilation action, alert, scene update |
| Light | Hub / ArtiOS rule processing | Lighting scene, curtain-related action where supported |
| Occupancy | Hub / ArtiOS rule processing | Scene activation, energy-saving action, alert |
| Environmental threshold | Hub / ArtiOS automation | Act / Sync / Panel / App notification |
| Sensor health event | Runtime health update | Device status update, alert, log |

## **中文表**

| **感測輸入**   | **Runtime 處理**      | **可能輸出**                     |
|----------------|-----------------------|----------------------------------|
| 溫度           | Hub / ArtiOS 規則處理 | Sync 空調指令、Act 控制、警報    |
| 濕度           | Hub / ArtiOS 規則處理 | 通風動作、警報、場景更新         |
| 光照           | Hub / ArtiOS 規則處理 | 照明場景、支援條件下窗簾相關動作 |
| 人在           | Hub / ArtiOS 規則處理 | 場景啟動、節能動作、警報         |
| 環境閾值       | Hub / ArtiOS 自動化   | Act / Sync / Panel / App 通知    |
| 感測器健康事件 | Runtime 健康更新      | 設備狀態更新、警報、日誌         |

## **8.7 Vision Event Flow**

## **8.7 視覺事件流**

**EN  
**Vision Event Flow defines how visual events from Artibird Vision enter
the system. Visual events may include motion events, visual alerts,
local recording status, security awareness signals, or room-state
signals where supported.

Visual events may be processed locally by Hub and ArtiOS, synchronized
with OceanAI Cloud where supported, or sent to OceanAI Edge for
AI-assisted visual interpretation where deployed. Vision data behavior
must follow privacy, storage, access, and validation boundaries.

**ZH-TW  
**視覺事件流定義來自 Artibird Vision
的視覺事件如何進入系統。視覺事件可包括移動事件、視覺警報、本地錄影狀態、安全感知信號，或受支援條件下的房間狀態信號。

視覺事件可由 Hub 與 ArtiOS 本地處理，在支援條件下與 OceanAI Cloud
同步，或在部署條件下送至 OceanAI Edge 進行 AI 輔助視覺解讀。Vision
數據行為必須遵循隱私、儲存、存取與驗證邊界。

## **8.8 Cloud Command Flow**

## **8.8 雲端指令流**

**EN  
**Cloud Command Flow defines how remote commands from OceanAI Cloud
enter the local system. A cloud command may originate from Artibird App
remote access, account-based control, cloud automation, lifecycle
service, notification action, or multi-site management.

Cloud commands should be routed to Artibird Hub and then to Artibird
Gate or target devices where supported. Cloud command execution depends
on network availability, account binding, permissions, Hub availability,
device connectivity, and validation status.

**ZH-TW  
**雲端指令流定義來自 OceanAI Cloud
的遠端指令如何進入本地系統。雲端指令可來自 Artibird App
遠端存取、基於帳號的控制、雲端自動化、生命週期服務、通知動作或多站點管理。

雲端指令應在支援條件下路由至 Artibird Hub，再到 Artibird Gate
或目標設備。雲端指令執行取決於網路可用性、帳號綁定、權限、Hub
可用性、設備連接與驗證狀態。

## **8.9 Edge Intelligence Flow**

## **8.9 邊緣智慧流**

**EN  
**Edge Intelligence Flow defines how selected sensor data, visual
context, device state, or runtime information may be sent to OceanAI
Edge for AI-assisted interpretation where deployed.

OceanAI Edge may return interpretation, recommendation, classification,
or context signals to Hub and ArtiOS. These outputs may inform
automation, alerts, energy behavior, security response, or scene
behavior where supported. Edge-assisted output must remain within
runtime priority, safety, privacy, and validation boundaries.

**ZH-TW  
**邊緣智慧流定義特定感測數據、視覺上下文、設備狀態或 Runtime
資訊如何在部署條件下送至 OceanAI Edge 進行 AI 輔助解讀。

OceanAI Edge 可向 Hub 與 ArtiOS
返回解讀、建議、分類或上下文信號。這些輸出可在支援條件下影響自動化、警報、能源行為、安全回應或場景行為。Edge
輔助輸出必須保持在 Runtime 優先級、安全、隱私與驗證邊界內。

## **8.10 Sync Flow**

## **8.10 同步流**

**EN  
**Sync Flow defines how selected state, configuration, lifecycle data,
logs, historical records, account information, and system status may
synchronize between Artibird Hub and OceanAI Cloud.

Synchronization may be continuous, scheduled, event-triggered, or manual
depending on system configuration. Sync behavior depends on network
availability, cloud availability, account binding, privacy policy,
storage limits, and validation conditions.

**ZH-TW  
**同步流定義特定狀態、配置、生命週期數據、日誌、歷史記錄、帳號資訊與系統狀態如何在
Artibird Hub 與 OceanAI Cloud 之間同步。

同步可依系統配置採用連續、定時、事件觸發或手動方式。同步行為取決於網路可用性、雲端可用性、帳號綁定、隱私政策、儲存限制與驗證條件。

## **English Table**

| **Sync Data Type** | **Direction** | **Description** |
|----|----|----|
| Device state | Hub ↔ Cloud | Selected device states may synchronize |
| Configuration | Hub ↔ Cloud | Device, room, scene, and automation configuration may synchronize |
| Lifecycle data | Hub ↔ Cloud | Firmware, software, and device lifecycle information may synchronize |
| Logs | Hub → Cloud | Selected runtime, diagnostic, and security logs may synchronize where supported |
| Historical records | Hub → Cloud | Historical data may synchronize where enabled |
| Account information | Cloud → Hub / App | User, role, and binding information may synchronize |
| Alerts | Hub / Cloud → App | Event alerts may be delivered through supported channels |

## **中文表**

| **同步數據類型** | **方向** | **說明** |
|----|----|----|
| 設備狀態 | Hub ↔ Cloud | 特定設備狀態可同步 |
| 配置 | Hub ↔ Cloud | 設備、房間、場景與自動化配置可同步 |
| 生命週期數據 | Hub ↔ Cloud | 韌體、軟體與設備生命週期資訊可同步 |
| 日誌 | Hub → Cloud | 支援條件下特定 Runtime、診斷與安全日誌可同步 |
| 歷史記錄 | Hub → Cloud | 啟用條件下歷史數據可同步 |
| 帳號資訊 | Cloud → Hub / App | 用戶、角色與綁定資訊可同步 |
| 警報 | Hub / Cloud → App | 事件警報可透過受支援通道傳送 |

## **8.12 Data Flow Boundary**

## **8.12 數據流邊界**

**EN  
**Data and event flow must be described according to supported product
type, device capability, firmware version, software version, network
condition, cloud availability, edge deployment status, privacy policy,
and validation evidence.

A defined flow does not imply that all data types, all devices, all
third-party services, all cloud behaviors, all edge models, or all
automation paths are supported.

**ZH-TW  
**數據與事件流必須依據受支援產品類型、設備能力、韌體版本、軟體版本、網路條件、雲端可用性、Edge
部署狀態、隱私政策與驗證證據進行描述。

已定義流程不表示支援所有數據類型、所有設備、所有第三方服務、所有雲端行為、所有
Edge 模型或所有自動化路徑。

## **8.13 Validation Requirements**

## **8.13 驗證要求**

**EN  
**Any data or event flow that supports a performance, reliability,
capacity, security, privacy, AI, or commercial claim must be validated
under defined conditions.

Validation should include source component, target component,
communication path, message type, expected behavior, failure behavior,
latency where applicable, and synchronization behavior where applicable.

**ZH-TW  
**任何支援性能、可靠性、容量、安全、隱私、AI
或商業宣稱的數據或事件流，都必須在明確條件下完成驗證。

驗證應包含來源組件、目標組件、通訊路徑、消息類型、預期行為、故障行為、適用時的延遲，以及適用時的同步行為。
