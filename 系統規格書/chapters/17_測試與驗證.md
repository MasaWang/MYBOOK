## **17.2 Testing and Validation Definition**

## **17.2 測試與驗證定義**

**EN  
**Testing and Validation define how the OceanAI / Artibird system is
verified under defined hardware, firmware, software, network, cloud,
edge, runtime, deployment, and environmental conditions.

Testing confirms whether supported functions operate as expected.
Validation confirms whether system claims, capacity limits, reliability
behavior, security behavior, privacy behavior, cloud participation, edge
participation, and deployment models are supported by evidence.

**ZH-TW  
**測試與驗證定義 OceanAI / Artibird
系統如何在明確硬體、韌體、軟體、網路、雲端、Edge、Runtime、部署與環境條件下完成確認。

測試用於確認受支援功能是否按預期運作。驗證用於確認系統宣稱、容量限制、可靠性行為、安全行為、隱私行為、雲端參與、Edge
參與與部署模型是否具備證據支持。

## **17.4 Functional Test**

## **17.4 功能測試**

**EN  
**Functional Test verifies whether the core system functions operate as
expected. It should cover device onboarding, local control, remote
control where supported, scene execution, automation execution, sensor
reporting, Vision event reporting, Act execution, Sync command behavior,
App interaction, Hub services, Gate communication, Cloud participation,
and Edge participation where applicable.

Functional testing must be performed according to supported product
type, firmware version, software version, deployment model, and expected
system behavior.

**ZH-TW  
**功能測試驗證核心系統功能是否按預期運作。它應涵蓋設備入網、本地控制、支援條件下遠端控制、場景執行、自動化執行、感測上報、Vision
事件上報、Act 執行、Sync 指令行為、App 互動、Hub 服務、Gate 通訊、Cloud
參與，以及適用時 Edge 參與。

功能測試必須依受支援產品類型、韌體版本、軟體版本、部署模型與預期系統行為進行。

## **English Table**

| **Functional Test Item** | **Expected Verification** |
|----|----|
| Device onboarding | Device can be discovered, registered, and assigned |
| Local control | Supported local commands execute correctly |
| Remote control | Supported remote commands execute under cloud conditions |
| Scene execution | Configured scenes execute expected device actions |
| Automation execution | Rules trigger and execute as configured |
| Sensor reporting | Sense data reports correctly |
| Vision event reporting | Vision events report correctly where supported |
| Act execution | Relay, plug, or load control executes correctly |
| Sync command behavior | IR / RF command behavior works where supported |
| App interaction | Setup, control, configuration, and status display operate correctly |

## **中文表**

| **功能測試項目** | **預期驗證**                       |
|------------------|------------------------------------|
| 設備入網         | 設備可被發現、註冊與分配           |
| 本地控制         | 受支援本地指令正確執行             |
| 遠端控制         | 在雲端條件下受支援遠端指令正確執行 |
| 場景執行         | 已配置場景執行預期設備動作         |
| 自動化執行       | 規則按配置觸發與執行               |
| 感測上報         | Sense 數據正確上報                 |
| Vision 事件上報  | 支援條件下 Vision 事件正確上報     |
| Act 執行         | 繼電器、插座或負載控制正確執行     |
| Sync 指令行為    | 支援條件下 IR / RF 指令行為正常    |
| App 互動         | 設定、控制、配置與狀態顯示正常運作 |

## **17.5 Runtime Test**

## **17.5 Runtime 測試**

**EN  
**Runtime Test verifies local runtime behavior managed by Artibird Hub
and ArtiOS. It should cover local service hosting, event handling, state
management, command routing, scene execution, automation execution, data
synchronization, offline behavior, and Edge participation where
deployed.

Runtime testing must verify that the system follows defined priority
rules, permission rules, supported device capability, and expected
failure behavior.

**ZH-TW  
**Runtime 測試驗證由 Artibird Hub 與 ArtiOS 管理的本地 Runtime
行為。它應涵蓋本地服務承載、事件處理、狀態管理、指令路由、場景執行、自動化執行、數據同步、離線行為，以及部署條件下
Edge 參與。

Runtime
測試必須驗證系統是否遵循已定義優先級規則、權限規則、受支援設備能力與預期故障行為。

## **English Table**

| **Runtime Test Item** | **Expected Verification** |
|----|----|
| Local service hosting | Hub services and ArtiOS modules operate correctly |
| Event handling | Events are processed by runtime logic |
| State management | Device and system states update correctly |
| Command routing | Commands reach intended targets |
| Scene execution | Runtime executes multi-device scenes correctly |
| Automation execution | Runtime executes configured rules correctly |
| Priority handling | Safety, manual control, automation, Edge, Cloud, and background priorities apply |
| Offline behavior | Supported local functions continue under defined offline conditions |
| Edge participation | Edge-assisted output informs runtime only within defined boundaries |

## **中文表**

| **Runtime 測試項目** | **預期驗證** |
|----|----|
| 本地服務承載 | Hub 服務與 ArtiOS 模組正常運作 |
| 事件處理 | 事件由 Runtime 邏輯正確處理 |
| 狀態管理 | 設備與系統狀態正確更新 |
| 指令路由 | 指令到達預期目標 |
| 場景執行 | Runtime 正確執行多設備場景 |
| 自動化執行 | Runtime 正確執行已配置規則 |
| 優先級處理 | 安全、手動控制、自動化、Edge、Cloud 與背景優先級正確適用 |
| 離線行為 | 在明確離線條件下受支援本地功能繼續 |
| Edge 參與 | Edge 輔助輸出僅在已定義邊界內影響 Runtime |

## **17.6 Connectivity Test**

## **17.6 連接測試**

**EN  
**Connectivity Test verifies network and gateway communication among
Hub, Gate, supported devices, App, Cloud, and Edge. It should cover
Ethernet, Wi-Fi, BLE Mesh, Wi-Fi Mesh where implemented, local MQTT, App
local connection, cloud connection, gateway communication, and offline
local behavior.

Connectivity testing must define network topology, device count,
firmware version, hardware version, signal condition, message rate, and
expected communication behavior.

**ZH-TW  
**連接測試驗證 Hub、Gate、受支援設備、App、Cloud 與 Edge
之間的網路與網關通訊。它應涵蓋 Ethernet、Wi-Fi、BLE Mesh、已實作條件下
Wi-Fi Mesh、本地 MQTT、App 本地連接、雲端連接、網關通訊與離線本地行為。

連接測試必須定義網路拓撲、設備數量、韌體版本、硬體版本、信號條件、消息速率與預期通訊行為。

## **English Table**

| **Connectivity Test Item** | **Expected Verification** |
|----|----|
| Ethernet connection | Wired connection operates where supported |
| Wi-Fi connection | Wireless connection operates under defined conditions |
| BLE Mesh behavior | Supported BLE Mesh devices communicate correctly |
| Wi-Fi Mesh behavior | Wi-Fi Mesh operates only where implemented and validated |
| Local MQTT | Broker, topics, messages, authentication, and routing operate correctly |
| App local connection | App discovers and controls local system where supported |
| Cloud connection | Cloud access and synchronization operate under defined conditions |
| Gate communication | Gate passes supported device events and commands correctly |
| Offline local network | Local network behavior continues without Internet where supported |

## **中文表**

| **連接測試項目** | **預期驗證**                                |
|------------------|---------------------------------------------|
| Ethernet 連接    | 支援條件下有線連接正常運作                  |
| Wi-Fi 連接       | 在明確條件下無線連接正常運作                |
| BLE Mesh 行為    | 受支援 BLE Mesh 設備正確通訊                |
| Wi-Fi Mesh 行為  | Wi-Fi Mesh 僅在已實作與驗證條件下運作       |
| 本地 MQTT        | Broker、Topic、消息、身份認證與路由正常運作 |
| App 本地連接     | App 在支援條件下發現並控制本地系統          |
| 雲端連接         | 雲端存取與同步在明確條件下正常運作          |
| Gate 通訊        | Gate 正確傳遞受支援設備事件與指令           |
| 離線本地網路     | 無 Internet 時支援的本地網路行為繼續        |

## **17.7 Capacity Test**

## **17.7 容量測試**

**EN  
**Capacity Test verifies measurable system limits. It should validate
max concurrent device connections, max devices per Gate, max devices per
Hub, max scenes, max automations, max rooms / spaces, max users, max
sites, MQTT throughput, event latency, storage limits, Vision stream
limits, and Edge processing limits.

Capacity testing must be performed under defined hardware, firmware,
software, network, cloud, edge, runtime load, and deployment conditions.

**ZH-TW  
**容量測試驗證可量測系統限制。它應驗證最大併發設備連接數、每個 Gate
最大設備數、每個 Hub 最大設備數、最大場景數、最大自動化數、最大房間 /
空間數、最大用戶數、最大站點數、MQTT 吞吐量、事件延遲、儲存限制、Vision
視頻流限制與 Edge 處理限制。

容量測試必須在明確硬體、韌體、軟體、網路、雲端、Edge、Runtime
負載與部署條件下執行。

## **English Table**

| **Capacity Test Item** | **Expected Verification** |
|----|----|
| Max concurrent device connections | Total active devices remain stable under defined load |
| Max devices per Gate | Gate maintains communication within validated capacity |
| Max devices per Hub | Hub maintains runtime behavior within validated capacity |
| Max scenes | Supported scene count operates correctly |
| Max automations | Supported automation count operates correctly |
| MQTT throughput | Message rate and latency meet acceptance criteria |
| Event latency | Source-to-action time meets expected requirement |
| Storage limits | Storage behavior follows defined retention and overflow rules |
| Vision stream limits | Vision streams operate within validated limits |
| Edge processing limits | Edge inference and processing remain within validated boundaries |

## **中文表**

| **容量測試項目**     | **預期驗證**                        |
|----------------------|-------------------------------------|
| 最大併發設備連接數   | 總活躍設備在明確負載下保持穩定      |
| 每個 Gate 最大設備數 | Gate 在已驗證容量內維持通訊         |
| 每個 Hub 最大設備數  | Hub 在已驗證容量內維持 Runtime 行為 |
| 最大場景數           | 支援場景數量正確運作                |
| 最大自動化數         | 支援自動化數量正確運作              |
| MQTT 吞吐量          | 消息速率與延遲符合驗收標準          |
| 事件延遲             | 從來源到動作的時間符合預期要求      |
| 儲存限制             | 儲存行為遵循已定義保留與溢出規則    |
| Vision 視頻流限制    | Vision 視頻流在已驗證限制內運作     |
| Edge 處理限制        | Edge 推理與處理保持在已驗證邊界內   |

## **17.8 Stress Test**

## **17.8 壓力測試**

**EN  
**Stress Test verifies long-duration operation and system behavior under
sustained or peak load. It should test device connections, message rate,
automation triggers, scene execution, cloud synchronization, Edge
processing, storage growth, network congestion, and recovery behavior.

Stress testing should identify system degradation, overload behavior,
memory growth, service instability, message delay, connection loss, and
recovery behavior.

**ZH-TW  
**壓力測試驗證長時間運作與系統在持續或峰值負載下的行為。它應測試設備連接、消息速率、自動化觸發、場景執行、雲端同步、Edge
處理、儲存增長、網路擁塞與恢復行為。

壓力測試應識別系統降級、過載行為、記憶體增長、服務不穩定、消息延遲、連接丟失與恢復行為。

## **English Table**

| **Stress Test Item** | **Expected Verification** |
|----|----|
| Long-duration runtime | System remains stable over defined test duration |
| High event rate | Runtime processes peak event load |
| High device count | System remains stable with defined device count |
| Repeated scene execution | Scenes execute repeatedly without abnormal behavior |
| Repeated automation triggers | Automations execute under frequent trigger conditions |
| Cloud sync load | Synchronization remains stable or degrades as defined |
| Edge processing load | Edge behavior remains within defined limits |
| Storage growth | Storage behavior follows retention and overflow policy |
| Network congestion | System degrades according to defined behavior |

## **中文表**

| **壓力測試項目** | **預期驗證**                 |
|------------------|------------------------------|
| 長時間 Runtime   | 系統在明確測試時長內保持穩定 |
| 高事件速率       | Runtime 能處理峰值事件負載   |
| 高設備數量       | 系統在明確設備數量下保持穩定 |
| 重複場景執行     | 場景反覆執行無異常行為       |
| 重複自動化觸發   | 自動化在高頻觸發條件下執行   |
| 雲端同步負載     | 同步保持穩定或按定義降級     |
| Edge 處理負載    | Edge 行為保持在已定義限制內  |
| 儲存增長         | 儲存行為遵循保留與溢出策略   |
| 網路擁塞         | 系統依已定義行為降級         |

## **17.9 Failover Test**

## **17.9 故障切換測試**

**EN  
**Failover Test verifies how the system behaves when cloud, Internet,
Hub, Gate, devices, Edge, App, power, local network, or storage becomes
unavailable or degraded.

Failover testing should verify expected degraded operation, supported
local continuation, unavailable functions, recovery behavior, status
reporting, alert behavior, and command handling.

**ZH-TW  
**故障切換測試驗證當雲端、Internet、Hub、Gate、設備、Edge、App、電力、本地網路或儲存不可用或降級時，系統如何運作。

故障切換測試應驗證預期降級運作、受支援本地延續、不可用功能、恢復行為、狀態回報、警報行為與指令處理。

## **English Table**

| **Failover Scenario** | **Expected Verification** |
|----|----|
| Cloud unavailable | Local supported functions continue where applicable |
| Internet unavailable | Local network behavior continues where configured |
| Hub unavailable | Hub-dependent runtime behavior stops or degrades as defined |
| Gate unavailable | Gate-dependent device access degrades as defined |
| Device offline | Device is marked offline and commands follow defined behavior |
| Edge unavailable | Edge-assisted functions stop or degrade while non-edge runtime continues |
| App unavailable | Physical controls and local automation continue where supported |
| Power failure | Recovery follows defined startup and state behavior |
| Network congestion | System degrades according to capacity and priority rules |

## **中文表**

| **故障切換場景** | **預期驗證**                                  |
|------------------|-----------------------------------------------|
| 雲端不可用       | 適用時受支援本地功能繼續                      |
| Internet 不可用  | 已配置條件下本地網路行為繼續                  |
| Hub 不可用       | Hub 依賴 Runtime 行為按定義停止或降級         |
| Gate 不可用      | 依賴 Gate 的設備接入按定義降級                |
| 設備離線         | 設備被標記離線，指令遵循已定義行為            |
| Edge 不可用      | Edge 輔助功能停止或降級，非 Edge Runtime 繼續 |
| App 不可用       | 支援條件下實體控制與本地自動化繼續            |
| 電力故障         | 恢復遵循已定義啟動與狀態行為                  |
| 網路擁塞         | 系統依容量與優先級規則降級                    |

## **17.10 Security Test**

## **17.10 安全測試**

**EN  
**Security Test verifies device identity, account binding, local access
control, remote access control, secure MQTT, firmware update security,
role-based access control, audit logs, and security-sensitive failure
behavior.

Security testing must verify both allowed behavior and rejected
behavior. Unauthorized access, invalid binding, invalid command,
unsupported device participation, invalid firmware, permission
violation, and insecure communication behavior must be tested where
applicable.

**ZH-TW  
**安全測試驗證設備身份、帳號綁定、本地存取控制、遠端存取控制、安全
MQTT、韌體更新安全、角色權限控制、審計日誌與安全敏感故障行為。

安全測試必須同時驗證允許行為與拒絕行為。未授權存取、無效綁定、無效指令、未支援設備參與、無效韌體、權限違反與不安全通訊行為，均應在適用條件下測試。

## **English Table**

| **Security Test Item** | **Expected Verification** |
|----|----|
| Device identity | Device identity is verified before participation |
| Account binding | Binding follows permission and security rules |
| Local access control | Local commands follow permission rules |
| Remote access control | Remote access requires valid authentication and authorization |
| Secure MQTT | MQTT authentication and protection work where implemented |
| Firmware update security | Invalid or unauthorized updates are rejected |
| Role-based access control | Roles limit access as defined |
| Audit logs | Security-sensitive events are recorded where supported |
| Unauthorized access | Unauthorized behavior is rejected or blocked |

## **中文表**

| **安全測試項目** | **預期驗證**                     |
|------------------|----------------------------------|
| 設備身份         | 設備參與前完成身份驗證           |
| 帳號綁定         | 綁定遵循權限與安全規則           |
| 本地存取控制     | 本地指令遵循權限規則             |
| 遠端存取控制     | 遠端存取需要有效身份認證與授權   |
| 安全 MQTT        | 已實作條件下 MQTT 認證與保護正常 |
| 韌體更新安全     | 無效或未授權更新被拒絕           |
| 角色權限控制     | 角色依定義限制存取               |
| 審計日誌         | 支援條件下安全敏感事件被記錄     |
| 未授權存取       | 未授權行為被拒絕或阻止           |

## **17.11 Privacy Test**

## **17.11 隱私測試**

**EN  
**Privacy Test verifies data collection, storage, processing,
synchronization, retention, deletion, access permission, Vision data
behavior, cloud data policy, and Edge processing boundary.

Privacy testing should confirm that sensitive data behavior follows user
permission, privacy settings, storage rules, cloud synchronization
rules, Edge processing rules, and documented data policies.

**ZH-TW  
**隱私測試驗證數據收集、儲存、處理、同步、保留、刪除、存取權限、Vision
數據行為、雲端數據政策與 Edge 處理邊界。

隱私測試應確認敏感數據行為符合用戶權限、隱私設定、儲存規則、雲端同步規則、Edge
處理規則與已記錄數據政策。

## **English Table**

| **Privacy Test Item** | **Expected Verification** |
|----|----|
| Data collection | Only defined data is collected |
| Data storage | Data is stored according to policy |
| Data retention | Retention follows defined duration and rules |
| Data deletion | Deletion or removal behavior works where supported |
| Vision privacy | Recording, storage, access, and sync follow privacy settings |
| Cloud data behavior | Cloud storage and use follow documented policy |
| Edge processing boundary | Edge processing follows local and privacy configuration |
| Access permission | Users can access only permitted data |
| Privacy restriction | Restricted data is not processed or shared outside defined rules |

## **中文表**

| **隱私測試項目** | **預期驗證**                       |
|------------------|------------------------------------|
| 數據收集         | 僅收集已定義數據                   |
| 數據儲存         | 數據依政策儲存                     |
| 數據保留         | 保留遵循已定義時長與規則           |
| 數據刪除         | 支援條件下刪除或移除行為正常       |
| Vision 隱私      | 錄影、儲存、存取與同步遵循隱私設定 |
| 雲端數據行為     | 雲端儲存與使用遵循已記錄政策       |
| Edge 處理邊界    | Edge 處理遵循本地與隱私配置        |
| 存取權限         | 用戶只能存取被允許數據             |
| 隱私限制         | 受限制數據不在規則之外被處理或共享 |

## **17.12 Compatibility Test**

## **17.12 相容性測試**

**EN  
**Compatibility Test verifies interoperability between supported
devices, firmware versions, hardware versions, software versions, App
versions, Hub versions, Gate versions, Cloud services, Edge services,

Compatibility testing must confirm supported combinations and identify

## **17.15 Testing Boundary**

## **17.15 測試邊界**

**EN  
**Testing and validation must be described according to defined test
conditions. A successful test under one hardware version, firmware
version, software version, network topology, device count, deployment
model, or cloud / edge condition does not automatically validate all
other conditions.

Testing results must not be generalized beyond the tested scope unless
additional validation evidence is available.

**ZH-TW  
**測試與驗證必須依明確測試條件描述。在某一硬體版本、韌體版本、軟體版本、網路拓撲、設備數量、部署模型或雲端
/ Edge 條件下測試成功，不自動代表其他所有條件已完成驗證。

除非具備額外驗證證據，否則測試結果不得超出已測範圍進行泛化。
