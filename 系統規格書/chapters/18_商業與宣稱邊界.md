# **Chapter 18 商業與宣稱邊界**

# **18. Commercial and Claim Boundaries**

# **18. 商業與宣稱邊界**

## **18.1 Purpose**

## **18.1 目的**

**EN  
**This section prevents system-level overclaims and ensures that
commercial language remains aligned with engineering validation.

**ZH-TW  
**本章防止系統級過度宣稱，並確保商業語言與工程驗證保持一致。

## **18.2 Commercial and Claim Boundaries Definition**

## **18.2 商業與宣稱邊界定義**

**EN  
**Commercial and Claim Boundaries define how the OceanAI / Artibird
system may be described in product documents, commercial specifications,
websites, product catalogs, channel materials, installation guides,
sales presentations, and customer-facing communication.

All commercial claims must remain aligned with supported system
capabilities, product specifications, engineering validation, deployment
conditions, security and privacy boundaries, certification status, and
test evidence.

**ZH-TW  
**商業與宣稱邊界定義 OceanAI / Artibird
系統可如何被描述於產品文件、商業規格書、網站、產品型錄、渠道材料、安裝指南、銷售簡報與面向客戶的溝通內容中。

所有商業宣稱都必須與受支援系統能力、產品規格書、工程驗證、部署條件、安全與隱私邊界、認證狀態與測試證據保持一致。

## **18.4 System-Level Claim Principle**

## **18.4 系統級宣稱原則**

**EN  
**System-level claims must describe the OceanAI / Artibird system
according to actual supported architecture, runtime behavior, product
capability, local-first operation, cloud-enhanced behavior,
edge-assisted intelligence, validated capacity, and defined deployment
conditions.

Commercial language may simplify technical content for readability, but
it must not create unsupported assumptions about compatibility,
capacity, reliability, AI capability, offline behavior, security,
privacy, certification, or deployment scale.

**ZH-TW  
**系統級宣稱必須依據實際受支援架構、Runtime
行為、產品能力、本地優先運作、雲端增強行為、Edge
輔助智慧、已驗證容量與已定義部署條件來描述 OceanAI / Artibird 系統。

商業語言可以為了可讀性簡化技術內容，但不得造成對相容性、容量、可靠性、AI
能力、離線行為、安全、隱私、認證或部署規模的未支援假設。

## **18.5 Product Compatibility Claims**

## **18.5 產品相容性宣稱**

**EN  
**Product compatibility claims must be limited to supported and
validated Artibird products, versions, communication paths, firmware
versions, software versions, and deployment models.

The system must not be described as universally compatible with all
third-party devices, all smart home platforms, all protocols, all
appliances, all routers, all electrical loads, or all installation
environments unless such compatibility has been validated and
documented.

**ZH-TW  
**產品相容性宣稱必須限制於受支援且已驗證的 Artibird
產品、版本、通訊路徑、韌體版本、軟體版本與部署模型。

除非相關相容性已完成驗證並形成文件，否則不得將系統描述為與所有第三方設備、所有智慧家居平台、所有協議、所有家電、所有路由器、所有電氣負載或所有安裝環境通用相容。

## **English Table**

| **Claim Area** | **Acceptable Claim Boundary** |
|----|----|
| Artibird compatibility | Supported and validated Artibird products |
| Third-party device compatibility | Only where tested and documented |
| Protocol compatibility | Only supported protocols and versions |
| Appliance compatibility | Only supported appliance categories and models where validated |
| Router compatibility | General router use may be stated only within tested network conditions |
| Platform compatibility | Only integrated and validated platforms |
| Electrical load compatibility | Only supported load types and ratings defined by product specifications |

## **中文表**

| **宣稱領域**     | **可接受宣稱邊界**                         |
|------------------|--------------------------------------------|
| Artibird 相容性  | 受支援且已驗證的 Artibird 產品             |
| 第三方設備相容性 | 僅限已測試並記錄的情況                     |
| 協議相容性       | 僅限受支援協議與版本                       |
| 家電相容性       | 僅限已驗證的受支援家電類別與型號           |
| 路由器相容性     | 僅可在已測網路條件內描述一般路由器使用     |
| 平台相容性       | 僅限已整合且已驗證平台                     |
| 電氣負載相容性   | 僅限產品規格書定義的受支援負載類型與額定值 |

## **18.6 Capacity Claims**

## **18.6 容量宣稱**

**EN  
**Capacity claims must be based on validated test reports under defined
hardware, firmware, software, network, cloud, edge, and deployment
conditions.

Any statement involving concurrent device connections, devices per Gate,
devices per Hub, MQTT throughput, event latency, number of scenes,
number of automations, Vision stream count, Edge processing load, users,
rooms, spaces, or sites must include or reference validated conditions.

**ZH-TW  
**容量宣稱必須基於明確硬體、韌體、軟體、網路、雲端、Edge
與部署條件下的已驗證測試報告。

任何涉及併發設備連接、每個 Gate 設備數、每個 Hub 設備數、MQTT
吞吐量、事件延遲、場景數、自動化數、Vision 視頻流數量、Edge
處理負載、用戶、房間、空間或站點的表述，都必須包含或引用已驗證條件。

## **English Table**

| **Capacity Claim Type** | **Required Boundary** |
|----|----|
| Max concurrent devices | Must state validated device count and test condition |
| Max devices per Gate | Must state Gate model, firmware version, protocol, and test condition |
| Max devices per Hub | Must state Hub hardware, service load, and test condition |
| MQTT throughput | Must state message rate, latency, and load condition |
| Event latency | Must state source, target, network path, and load condition |
| Vision streams | Must state Vision device count, stream type, storage, network, and processing condition |
| Edge processing | Must state hardware platform, model version, input type, and load condition |

## **中文表**

| **容量宣稱類型** | **必須邊界** |
|----|----|
| 最大併發設備數 | 必須說明已驗證設備數與測試條件 |
| 每個 Gate 最大設備數 | 必須說明 Gate 型號、韌體版本、協議與測試條件 |
| 每個 Hub 最大設備數 | 必須說明 Hub 硬體、服務負載與測試條件 |
| MQTT 吞吐量 | 必須說明消息速率、延遲與負載條件 |
| 事件延遲 | 必須說明來源、目標、網路路徑與負載條件 |
| Vision 視頻流 | 必須說明 Vision 設備數、視頻流類型、儲存、網路與處理條件 |
| Edge 處理 | 必須說明硬體平台、模型版本、輸入類型與負載條件 |

## **18.7 Local-First and Offline Claims**

## **18.7 本地優先與離線宣稱**

**EN  
**Local-first and offline claims must clearly define which functions
operate locally and which functions require cloud services.

The system may be described as local-first only when the statement
refers to supported local functions under required local component
conditions. It must not be described as fully offline for all functions
unless all referenced functions have been validated without Internet or
cloud access.

**ZH-TW  
**本地優先與離線宣稱必須清楚定義哪些功能可本地運作，哪些功能需要雲端服務。

只有當表述指向在所需本地元件條件下受支援本地功能時，才可將系統描述為本地優先。除非所引用的全部功能都已在無
Internet
或無雲端存取條件下完成驗證，否則不得描述為所有功能完全離線可用。

## **English Table**

| **Claim Type** | **Required Boundary** |
|----|----|
| Local-first control | Must define required local components |
| Offline scene execution | Must define local scene configuration and required devices |
| Offline automation | Must define local automation rules and input / output devices |
| Offline App control | Must define local network, App version, Hub availability, and permission condition |
| Cloud-independent behavior | Must specify which behavior does not require cloud |
| Remote access | Must be described as cloud-required |
| Cloud synchronization | Must be described as unavailable or delayed when cloud is unavailable |

## **中文表**

| **宣稱類型**   | **必須邊界**                                     |
|----------------|--------------------------------------------------|
| 本地優先控制   | 必須定義所需本地組件                             |
| 離線場景執行   | 必須定義本地場景配置與所需設備                   |
| 離線自動化     | 必須定義本地自動化規則與輸入 / 輸出設備          |
| 離線 App 控制  | 必須定義本地網路、App 版本、Hub 可用性與權限條件 |
| 不依賴雲端行為 | 必須說明哪些行為不需要雲端                       |
| 遠端存取       | 必須描述為需要雲端                               |
| 雲端同步       | 必須描述為雲端不可用時不可用或延遲               |

## **18.8 Cloud Claims**

## **18.8 雲端宣稱**

**EN  
**Cloud claims must be limited to implemented and supported OceanAI
Cloud services. Cloud services may include remote access,
synchronization, account services, alerts, lifecycle management,
historical data, analytics, multi-site management, and commercial
service plans where supported.

Cloud claims must not imply guaranteed cloud availability, unlimited
storage, unrestricted remote access, unsupported service levels, or
undefined data use.

**ZH-TW  
**雲端宣稱必須限制於已實作且受支援的 OceanAI Cloud
服務。雲端服務可在支援條件下包括遠端存取、同步、帳號服務、警報、生命週期管理、歷史數據、分析、多站點管理與商業服務方案。

雲端宣稱不得暗示保證雲端可用、無限制儲存、不受限制遠端存取、未支援服務等級或未定義數據使用。

## **English Table**

| **Cloud Claim Type** | **Required Boundary** |
|----|----|
| Remote access | Requires Internet, account binding, permission, and cloud availability |
| Cloud synchronization | Requires supported data type and cloud availability |
| Alerts | Requires supported event type, App permission, and notification condition |
| Historical data | Requires enabled service, storage policy, and retention rule |
| Multi-site management | Requires supported account or organization structure |
| Firmware lifecycle | Requires supported product, version, and update path |
| Cloud analytics | Requires supported data source and policy-defined data use |

## **中文表**

| **雲端宣稱類型** | **必須邊界**                              |
|------------------|-------------------------------------------|
| 遠端存取         | 需要 Internet、帳號綁定、權限與雲端可用性 |
| 雲端同步         | 需要受支援數據類型與雲端可用性            |
| 警報             | 需要受支援事件類型、App 權限與通知條件    |
| 歷史數據         | 需要已啟用服務、儲存政策與保留規則        |
| 多站點管理       | 需要受支援帳號或組織結構                  |
| 韌體生命週期     | 需要受支援產品、版本與更新路徑            |
| 雲端分析         | 需要受支援數據來源與政策定義數據使用      |

## **18.9 Edge and AI Claims**

## **18.9 Edge 與 AI 宣稱**

**EN  
**Edge and AI claims must be described as AI-assisted behavior under
defined conditions. Claims involving SLM, VLM, visual understanding,
sensor fusion, optimization, energy behavior, security interpretation,
or automation support must state the supported model, input type,
deployment condition, privacy boundary, and validation requirement.

The system must not be described as fully autonomous, perfectly
accurate, universally intelligent, or capable of unrestricted control
unless such behavior is explicitly implemented, validated, and bounded
by safety, permission, and runtime rules.

**ZH-TW  
**Edge 與 AI 宣稱必須描述為在明確條件下的 AI 輔助行為。任何涉及
SLM、VLM、視覺理解、感測融合、優化、能源行為、安全解讀或自動化支援的宣稱，都必須說明受支援模型、輸入類型、部署條件、隱私邊界與驗證要求。

不得將系統描述為完全自主、完美準確、通用智慧或可不受限制控制，除非此類行為已明確實作、驗證，並受到安全、權限與
Runtime 規則約束。

## **English Table**

| **Restricted AI Claim** | **Preferred Language** |
|----|----|
| Fully autonomous AI control | AI-assisted runtime support under defined conditions |
| Perfect visual understanding | Supports defined visual interpretation where validated |
| Guaranteed optimization | Provides AI-assisted optimization suggestions where validated |
| Understands every environment | Interprets supported input data under validated deployment conditions |
| Replaces user decisions | Assists runtime behavior while following permissions and control boundaries |
| Always improves energy efficiency | Supports energy-aware workflows where validated |
| Fully private AI by default | Privacy behavior follows configuration, policy, and validated processing boundary |

## **中文表**

| **限制 AI 宣稱** | **建議語言**                            |
|------------------|-----------------------------------------|
| 完全自主 AI 控制 | 在明確條件下提供 AI 輔助 Runtime 支援   |
| 完美視覺理解     | 在已驗證條件下支援明確視覺解讀          |
| 保證優化         | 在已驗證條件下提供 AI 輔助優化建議      |
| 理解所有環境     | 在已驗證部署條件下解讀受支援輸入數據    |
| 取代用戶決策     | 在遵循權限與控制邊界下輔助 Runtime 行為 |
| 永遠提升能源效率 | 在已驗證條件下支援能源感知工作流        |
| 預設完全私有 AI  | 隱私行為遵循配置、政策與已驗證處理邊界  |

## **18.10 Security and Privacy Claims**

## **18.10 安全與隱私宣稱**

**EN  
**Security and privacy claims must match implemented behavior,
documented data policies, permission rules, and validation evidence.

The system must not be described as absolutely secure, impossible to
breach, fully private under all conditions, or compliant with every
external regulation unless such claims are legally reviewed, technically
validated, and supported by proper documentation.

**ZH-TW  
**安全與隱私宣稱必須符合已實作行為、已記錄數據政策、權限規則與驗證證據。

除非此類宣稱已完成法律審查、技術驗證並具備適當文件支持，否則不得將系統描述為絕對安全、無法被攻破、所有條件下完全私密，或符合所有外部法規。

## **English Table**

| **Security / Privacy Claim** | **Required Boundary** |
|----|----|
| Secure device access | Must state implemented authentication and access control |
| Secure MQTT | Must state where authentication and encryption are implemented |
| Privacy protection | Must follow documented privacy settings and data policy |
| Vision privacy | Must define recording, storage, access, sync, and retention behavior |
| Edge privacy | Must define local processing and cloud sync boundary |
| Cloud data protection | Must follow cloud data storage, access, and retention policy |
| Role-based access | Must define supported roles and permission model |
| Audit logging | Must state supported log types and retention behavior |

## **中文表**

| **安全 / 隱私宣稱** | **必須邊界**                             |
|---------------------|------------------------------------------|
| 安全設備存取        | 必須說明已實作身份認證與存取控制         |
| 安全 MQTT           | 必須說明哪些條件下實作身份認證與加密     |
| 隱私保護            | 必須遵循已記錄隱私設定與數據政策         |
| Vision 隱私         | 必須定義錄影、儲存、存取、同步與保留行為 |
| Edge 隱私           | 必須定義本地處理與雲端同步邊界           |
| 雲端數據保護        | 必須遵循雲端數據儲存、存取與保留政策     |
| 角色權限            | 必須定義受支援角色與權限模型             |
| 審計日誌            | 必須說明受支援日誌類型與保留行為         |

## **18.11 Reliability Claims**

## **18.11 可靠性宣稱**

**EN  
**Reliability claims must be based on tested failure behavior, recovery
behavior, degraded operation, offline behavior, and capacity boundaries.

The system must not be described as never failing, never disconnecting,
always online, always cloud-available, or always functioning under all
network, power, product, and deployment conditions.

**ZH-TW  
**可靠性宣稱必須基於已測試故障行為、恢復行為、降級運作、離線行為與容量邊界。

不得將系統描述為永不故障、永不斷線、永遠在線、雲端永遠可用，或在所有網路、電力、產品與部署條件下永遠正常運作。

## **English Table**

| **Restricted Reliability Claim** | **Preferred Language** |
|----|----|
| Never disconnects | Designed for stable operation under validated conditions |
| Always online | Supports defined online and offline behavior |
| Always works during Internet outage | Supports defined local functions when required local components are present |
| No failure | Includes defined failure, degraded, and recovery behavior |
| Always recovers automatically | Recovery behavior depends on product, network, power, and system condition |
| Works in all environments | Works under supported deployment and environmental conditions |

## **中文表**

| **限制可靠性宣稱**      | **建議語言**                             |
|-------------------------|------------------------------------------|
| 永不斷線                | 在已驗證條件下設計用於穩定運作           |
| 永遠在線                | 支援已定義在線與離線行為                 |
| Internet 中斷時永遠可用 | 當所需本地元件具備時，支援已定義本地功能 |
| 無故障                  | 包含已定義故障、降級與恢復行為           |
| 永遠自動恢復            | 恢復行為取決於產品、網路、電力與系統條件 |
| 適用所有環境            | 在受支援部署與環境條件下運作             |

## **18.12 Energy Claims**

## **18.12 能源宣稱**

**EN  
**Energy-related claims must be based on validated measurement,
supported Act-E or energy-capable devices, supported data sources,
defined deployment conditions, and documented calculation method.

The system may be described as supporting energy-aware workflows where
validated. It must not be described as guaranteeing energy savings
unless supported by controlled test evidence and clearly defined
conditions.

**ZH-TW  
**能源相關宣稱必須基於已驗證量測、受支援 Act-E
或具備能源能力的設備、受支援數據來源、明確部署條件與已記錄計算方法。

在已驗證條件下，系統可描述為支援能源感知工作流。除非具備受控測試證據與明確條件，否則不得描述為保證節能。

## **English Table**

| **Energy Claim Type** | **Required Boundary** |
|----|----|
| Energy monitoring | Requires supported energy measurement device |
| Energy report | Requires defined data source and calculation method |
| Energy optimization | Must be described as assisted or rule-based unless validated otherwise |
| Energy saving | Must include tested scenario, baseline, method, and result |
| Load control | Must follow supported load type and rating |
| Energy alert | Requires supported threshold, measurement, and notification path |

## **中文表**

| **能源宣稱類型** | **必須邊界**                           |
|------------------|----------------------------------------|
| 能源監測         | 需要受支援能源計量設備                 |
| 能源報告         | 需要已定義數據來源與計算方法           |
| 能源優化         | 除非另有驗證，必須描述為輔助或基於規則 |
| 節能             | 必須包含已測場景、基準、方法與結果     |
| 負載控制         | 必須遵循受支援負載類型與額定值         |
| 能源警報         | 需要受支援閾值、量測與通知路徑         |

## **18.13 Certification Claims**

## **18.13 認證宣稱**

**EN  
**Certification claims must only be used after certification is complete
and supported by official documentation.

Product documents may state certification targets, certification
planning, or certification readiness only if the language clearly
indicates that certification is not yet complete. Certification claims
must not imply UL, ETL, FCC, CE, or other certification status before
approval is granted.

**ZH-TW  
**認證宣稱只能在認證完成並具備官方文件支持後使用。

如果認證尚未完成，產品文件可描述認證目標、認證規劃或認證準備狀態，但語言必須清楚表明認證尚未完成。在獲得核准之前，不得暗示已具備
UL、ETL、FCC、CE 或其他認證狀態。

## **English Table**

| **Certification Language** | **Usage Condition** |
|----|----|
| Certified | Use only after certification is complete |
| Listed | Use only when listing is officially granted |
| Compliant | Use only where compliance is verified and legally appropriate |
| Designed for certification | May be used for design intention if not misleading |
| Certification target | May be used when clearly identified as a target |
| Certification pending | May be used only if actually in process |
| Certification ready | Use carefully and only when supported by internal review |

## **中文表**

| **認證語言** | **使用條件**                       |

## **18.16 Claim Boundary**

**ZH-TW
