## **16.2 System Capacity and Engineering Boundaries Definition**

## **16.2 系統容量與工程邊界定義**

**EN  
**System Capacity and Engineering Boundaries define the measurable
limits of the OceanAI / Artibird system under supported hardware,
firmware, software, network, cloud, edge, and deployment conditions.

Capacity boundaries may include concurrent device connections, devices
per Gate, devices per Hub, scenes, automations, rooms, users, sites,
MQTT throughput, event latency, storage limits, Vision stream limits,
and Edge processing limits.

Engineering boundaries prevent unsupported claims and ensure that system
behavior is described according to validated test evidence rather than
theoretical capability.

**ZH-TW  
**系統容量與工程邊界定義 OceanAI / Artibird
系統在受支援硬體、韌體、軟體、網路、雲端、Edge
與部署條件下的可量測限制。

容量邊界可包括併發設備連接數、每個 Gate 支援設備數、每個 Hub
支援設備數、場景數、自動化數、房間數、用戶數、站點數、MQTT
吞吐量、事件延遲、儲存限制、Vision 視頻流限制與 Edge 處理限制。

工程邊界用於防止未支援宣稱，並確保系統行為依據已驗證測試證據描述，而不是依據理論能力描述。

## **16.4 Max Concurrent Device Connections**

## **16.4 最大併發設備連接數**

**EN  
**Max concurrent device connections define the total number of supported
active devices that may participate in the system at the same time under
validated conditions.

This capacity must be measured according to product type, network
topology, Gate configuration, Hub configuration, protocol type, message
rate, firmware version, software version, and deployment model.

Theoretical chip capability must not be used as the final system claim
unless the complete system has been tested under defined conditions.

**ZH-TW  
**最大併發設備連接數定義在已驗證條件下，可同時參與系統的受支援活躍設備總數。

此容量必須依產品類型、網路拓撲、Gate 配置、Hub
配置、協議類型、消息速率、韌體版本、軟體版本與部署模型進行量測。

除非完整系統已在明確條件下完成測試，否則不得將理論晶片能力作為最終系統宣稱。

## **English Table**

| **Validation Item** | **Required Definition** |
|----|----|
| Device count | Number of active devices under test |
| Product mix | Hub, Gate, Switch, Panel, Act, Sense, Sync, Vision mix |
| Network topology | Ethernet, Wi-Fi, BLE Mesh, local network, cloud path |
| Message rate | Event, command, telemetry, and synchronization rate |
| Runtime load | Scene, automation, state, and command processing load |
| Test duration | Duration of concurrent connection test |
| Failure condition | Behavior under disconnect, reconnect, congestion, or overload |
| Acceptance criteria | Required stability, latency, and error threshold |

## **中文表**

| **驗證項目** | **必須定義內容**                                        |
|--------------|---------------------------------------------------------|
| 設備數量     | 測試中的活躍設備數                                      |
| 產品組合     | Hub、Gate、Switch、Panel、Act、Sense、Sync、Vision 組合 |
| 網路拓撲     | Ethernet、Wi-Fi、BLE Mesh、本地網路、雲端路徑           |
| 消息速率     | 事件、指令、遙測與同步速率                              |

## **16.5 Max Devices per Gate**

## **16.5 每個 Gate 最大設備數**

**EN  
**Max devices per Gate defines the supported number of devices that may
connect through one Artibird Gate under tested network conditions.

This boundary depends on Gate hardware version, firmware version,
supported protocol, Wi-Fi condition, BLE Mesh condition, Wi-Fi Mesh
implementation where supported, device type, message frequency, and
deployment environment.

The capacity of Artibird Gate must be validated as a system-level
boundary, not only as a chipset-level capability.

**ZH-TW  
**每個 Gate 最大設備數定義在已測網路條件下，可透過一個 Artibird Gate
接入的受支援設備數量。

此邊界取決於 Gate 硬體版本、韌體版本、受支援協議、Wi-Fi 條件、BLE Mesh
條件、支援條件下 Wi-Fi Mesh 實作、設備類型、消息頻率與部署環境。

Artibird Gate 的容量必須作為系統級邊界進行驗證，而不僅是晶片級能力。

| **Gate Capacity Item** | **Required Definition** |

| Device type | Switch, Sense, Act, Panel, Vision, Sync, or mixed devices |
| Device count | Number of devices connected through one Gate |
| Event frequency | Average and peak event frequency |
| Gateway load | CPU, memory, message handling, connection handling where available |
| Failure behavior | Behavior when device count exceeds validated capacity |

## **中文表**

| Gate 型號         | Gate 產品型號與硬體版本                            |

吞吐量應依消息速率、負載大小、Topic 數量、客戶端數量、保留消息行為、QoS
行為、Broker 負載、Runtime 處理負載與明確條件下的事件延遲進行量測。

負載、設備數量、協議類型、雲端條件、Edge 條件與預期行為進行量測。
